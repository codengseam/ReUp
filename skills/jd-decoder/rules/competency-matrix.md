# 规则：能力矩阵权重表

> 本文件定义 jd-decoder 第 2 步的执行细则。返回上层路由见 `../SKILL.md`。理论根基见 `../references/jd-decoder-refs.md`。

基于 McClelland (1973) 胜任力理论与 Spencer & Spencer (1993) 冰山模型，将 JD 要求拆为显性 / 隐性两层胜任力，并给出权重计算方法。

## 胜任力分层模型（冰山模型映射）

| 层级 | 类型 | 冰山位置 | JD 对应内容 | 权重 |
|------|------|---------|------------|------|
| 知识 | 显性 | 冰上 | 学历、专业、行业知识、证书 | 显性合计 60% |
| 技能 | 显性 | 冰上 | 技术栈、工具、可观测的"会做" | 显性合计 60% |
| 社会角色 | 隐性 | 冰下 | 对角色的期待（如"对结果负责""跨团队协作"） | 隐性合计 40% |
| 自我认知 | 隐性 | 冰下 | 自我定位（如"以 owner 心态""主动驱动"） | 隐性合计 40% |
| 特质 | 隐性 | 冰下 | 性格倾向（如"抗压""细节敏感""拥抱变化"） | 隐性合计 40% |
| 动机 | 隐性 | 冰下 | 内驱力（如"追求卓越""创业心态""成就导向"） | 隐性合计 40% |

**核心原则**：显性胜任力可培训、易替换；隐性胜任力难培养、决定长期匹配。两类不可偏废，故默认权重 60/40（非 100/0）。

---

## 权重计算方法

### 第 1 步：归类
对 `./jd-structure-parsing.md` 产出的每条要求，按下表归类到 6 个胜任力层：

| JD 字段 | 归类去向 |
|---------|---------|
| hardRequirements.学历/证书 | 知识 |
| hardRequirements.经验 | 跨"知识+技能"（按经验描述拆：年限→知识，具体做过的事→技能） |
| hardRequirements.技能 | 技能 |
| skills（带程度词） | 技能 |
| responsibilities 中"负责/设计/优化" | 技能 |
| responsibilities 中"带领/管理/协调" | 社会角色 |
| responsibilities/要求中"owner/主动/自驱" | 自我认知 |
| 软性描述"抗压/拥抱变化/细节" | 特质 |
| 软性描述"创业心态/成就导向/追求卓越" | 动机 |

### 第 2 步：计数与归一
设显性层条目数 = N_visible，隐性层条目数 = N_hidden。

- 显性层内部按条目均分 60%：`知识权重 + 技能权重 = 60%`，二者按各自条目数比例切分。
- 隐性层内部按条目均分 40%：`社会角色 + 自我认知 + 特质 + 动机 = 40%`，四者按各自条目数比例切分。
- 某层无条目：该层权重为 0，其余层在所属类型内重新归一（保持 60/40 总比不变）。

### 第 3 步：硬性条件加权
`priority: "must"` 的条目权重 ×1.5（相对 preferred），归一化后反映"硬性条件在评估中权重更高"。

### 计算示例
JD 含：学历(must)、技能A(must)、技能B(preferred)、职责"带领团队"、软性"抗压"。
- 显性：知识(学历)=1 must、技能(技能A must + 技能B preferred + 经验归技能)=2，其中 1 must 1 preferred。
- 隐性：社会角色(带领)=1、特质(抗压)=1。
- 显性 must 条目加权：学历(1×1.5) + 技能A(1×1.5) + 技能B(1) = 4 等效单位，归一到 60%。
- 隐性均分：社会角色 20%、特质 20%。

输出示例片段：
```json
{
  "competency_matrix": {
    "visible": { "weight": 0.60, "layers": { "knowledge": 0.225, "skill": 0.375 } },
    "hidden":  { "weight": 0.40, "layers": { "social_role": 0.20, "trait": 0.20 } }
  }
}
```

## 硬性条件 vs 加分项区分

| 维度 | 硬性条件（must / 红线） | 加分项（preferred / 锦上添花） |
|------|----------------------|----------------------------|
| 来源标记 | `priority: "must"` | `priority: "preferred"` / `required: false` |
| 评估语义 | 不满足直接淘汰 | 不满足不淘汰，满足则加分 |
| 原文线索 | "必须/必备/硬性/及以上/至少" | "优先/加分/plus/nice-to-have/熟悉即可" |
| 矩阵权重 | ×1.5 加权 | ×1.0 |
| 输出标注 | `is_red_line: true` | `is_red_line: false` |

**红线汇总**：所有 must 条目单独汇总为 `red_lines` 数组，供下游匹配器（`src/features/jd/smart-matcher.ts`）做一票否决判断。

## 不同职级权重差异表

隐性胜任力的权重随职级上升而递增——高职级更看重"冰山下"的领导力与动机，低职级更看重"冰山上"的即战技能。

| 职级段位 | 显性权重 | 隐性权重 | 隐性层侧重 | 说明 |
|---------|---------|---------|-----------|------|
| 初级（P4/P5，0-3 年） | 75% | 25% | 特质 > 动机 | 即插即用为主，隐性只看"能不能扛事" |
| 中级（P6，3-5 年） | 65% | 35% | 社会角色 + 特质 | 开始要"协作 + owner 心态" |
| 高级（P7，5-8 年） | 55% | 45% | 社会角色 + 自我认知 | 要"跨团队影响 + 主动驱动" |
| 资深/专家（P8+，8 年+） | 45% | 55% | 动机 + 自我认知 | 要"创业心态 + 成就导向"，隐性反超显性 |
| 管理线（M2+） | 40% | 60% | 社会角色 + 动机 | 管理岗隐性占主导，技术深度让位于领导力 |

### 职级触发规则
- 职级段位由 `jinsheng-san-yuanze`（三原则映射）+ `./jd-structure-parsing.md` 字段 3 的 level 共同确定。
- 若 JD 未给职级：用 hardRequirements.经验年限估算（0-3→初级，3-5→中级，5-8→高级，8+→资深），并在输出中标注 `level_inferred: true`。
- 管理线判定：responsibilities 中"带领/管理/汇报/团队规模"出现 ≥ 2 次 → 管理线权重表。

### 权重覆盖规则
- 默认 60/40 仅在"无法确定职级"时使用。
- 一旦职级段位确定，以上表覆盖默认权重，并重新归一显性/隐性内部各层。
- 输出中必须同时给出 `default_or_level_based` 字段标明使用了哪套权重。

## 输出契约

本规则产出 `competency_matrix` 对象与 `red_lines` 数组，合并进 `../scripts/jd-profile-template.md` 的岗位画像。后续：
- `red_lines` → 供 `src/features/jd/smart-matcher.ts` 做一票否决。
- `competency_matrix.hidden` → 供 `./implicit-requirements.md` 做隐性推断交叉验证。
- 隐性层侧重 → 供 `./focus-points-inference.md` 决定行为面试维度权重。
