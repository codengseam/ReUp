# 应聘者素质模型对齐法

## 概念 (Concept)

### I — 方法论骨架 (Interpretation)

面试官评判候选人时并非只看表面的技术能力，而是基于一个四层递进的冰山模型：
1. **经验**：过去做过什么（工作对象和产出）。
2. **技能**：当前能做什么（解决问题的能力）。
3. **潜力**：未来能长多快（学习、创新、精益的能力）。
4. **动机**：为什么愿意做（底层价值观、驱动力、人品）。
普通的应聘者只在"经验和技能"层面对话，而高手懂得通过具体的事例向下深挖，主动向面试官证明自己的"潜力"和"动机"。因为即使技能暂时不匹配，只要潜力和动机足够强，依然能获得认可。

## 解释 (Explanation)

### E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **诊断当前回答的层次**
   - 听取用户的初步回答或经历描述。
   - 完成标准: 指出用户的回答目前停留在四层模型（经验、技能、潜力、动机）的哪一层。通常用户会停留在前两层。
   - **[检查点]**: 向用户展示诊断结果，并询问是否准备好深挖潜力层，暂停等待确认。

2. **挖掘潜力层（学习与创新）**
   - 引导用户补充：在做这件事的过程中，你是如何快速学习新知识的？遇到了什么困难并创新性地解决了？
   - **潜力层挖掘话术**：
     ```
     【学习能力】"这个项目中，你遇到过完全陌生的技术领域吗？你是怎么在多短时间内上手的？"
     【创新能力】"在完成这个任务时，你有没有发现并改进过什么低效的流程或工具？"
     【精益能力】"做完这个项目后，你有没有做过复盘？发现了哪些可以做得更好的地方？"
     ```
   - 完成标准: 提取出至少 1 个能证明"成长型思维"的具体细节。
   - **[检查点]**: 暂停等待用户回答。

3. **深究动机层（驱动力与价值观）**
   - 引导用户补充：做这件事时，除了为了完成任务，你内心的驱动力是什么？你对质量或团队有什么超出本职工作的追求？
   - **动机层挖掘话术**：
     ```
     【内在驱动】"是什么让你愿意在这个项目上投入比预期更多的时间？"
     【价值观体现】"当项目进度和质量冲突时，你是怎么权衡的？为什么？"
     【团队贡献】"你有没有主动帮助过团队成员解决问题？当时是怎么想的？"
     ```
   - 完成标准: 提炼出 1 个能展现"自燃型"或"高度责任感"的动机表达。
   - **[检查点]**: 暂停等待用户回答。

4. **重组回答结构**
   - 将用户的素材按照"经验铺垫 -> 技能展示 -> 潜力印证 -> 动机升华"的结构重写。
   - **四层递进回答模板**：
     ```
     【经验层】"在上一个项目中，我负责了XX模块的开发..."
     【技能层】"通过这个项目，我掌握了XX技术，解决了XX问题..."
     【潜力层】"在过程中，我主动学习了YY技术，并优化了ZZ流程，将效率提升了N%..."
     【动机层】"之所以愿意投入额外精力，是因为我坚信质量是产品的生命线..."
     ```
   - 完成标准: 输出一段层次分明、有血有肉的最终回答话术。

### B — 边界 (Boundary) ★

#### 不要在以下情况使用此 skill

- 遇到明确的算法题（如"翻转二叉树怎么写"）或八股文（如"TCP三次握手"）时。此时面试官在做纯"技能"探测，强行谈动机显得虚伪。

#### 作者在书中警告的失败模式

- **错把创业当打工**：在面试初创公司时过分强调职责边界，缺乏将创业当成事业的自燃型动机。
- **不懂装懂的诚信红线**：遇到超出自身知识边界的技术问题时强行掩饰，这会直接摧毁面试官对你"动机/价值观"层面的信任。

#### 作者的盲点 / 时代局限

- 随着行业内卷，初筛时企业对"即插即用的经验"看重程度极高。因此，不能完全抛弃"经验和技能"去空谈"潜力和动机"，四者必须相辅相成。

#### 容易混淆的邻近方法论

- STAR 法则（侧重于叙事结构，而本模型侧重于考查内核）。

## 场景 (Scenario)

### A1 — 书中的应用 (Past Application)

#### 案例 1: 跨专业无经验求职者的逆袭
- **问题**: 一个非计算机专业、无项目经验的应届生想应聘开发岗位，简历投递无回应。
- **方法论的使用**: 作者指出，在缺乏"经验"和"技能"时，必须用"潜力"和"动机"来弥补。对于基础弱的实习生，团队会刻意考察其学习能力和抗压能力。
- **结论**: 建议该候选人先通过自学做出成果展示学习能力，并展现出破釜沉舟的抗压动机。
- **结果**: 作者团队在确认某位实习生具备超强学习能力和抗压能力（高潜力和强动机）后，发放了转正 offer。

### A2 — 触发场景 (Future Trigger) ★

#### 用户会在什么情境下需要这个 skill?

1. 用户面临一个开放式的面试问题（如"说说你的优缺点"、"你最大的成就是什么"），不知道从何说起。
2. 用户觉得自己某项技术栈不匹配，担心面试通不过。
3. 用户在准备自我介绍，罗列了一堆流水账。

#### 语言信号 (用户的话里出现这些就应激活)

- "这个面试题考察的是什么？"
- "我没有这方面的经验，面试该怎么回答？"
- "帮我优化一下这段自我介绍"
- "我的回答是不是太干瘪了"

#### 与相邻 skill 的区分

- 与 `highlight-extractor` 的区别: 本 skill 侧重于宏观的**回答层次与立意拔高**（从技能切入动机），而 highlight-extractor 侧重于从日常琐事中**榨取具体的闪光点素材**。
