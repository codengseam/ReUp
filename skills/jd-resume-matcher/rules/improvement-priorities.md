# improvement-priorities.md — 改进建议优先级规则

> 被 `../SKILL.md` 第 8 步调用。本文件用"影响 × 可行性"矩阵把改进建议排为 P0-P3，每条建议必须指向简历具体段落。AI 不得给出无段落指向的泛泛建议。

## 1. 评估矩阵

建议优先级由"对匹配分的影响"与"修复可行性（时间成本）"两轴决定：

| | 高可行性（≤ 1 周） | 中可行性（1-4 周） | 低可行性（> 1 月 / 不可改） |
|---|---|---|---|
| **高影响** | **P0** | **P1** | **P3** |
| **中影响** | **P2** | **P1/P2** | **P3** |
| **低影响** | **P2** | **P2/P3** | **P3** |

### 1.1 影响判定

- 高影响：直接作用于 Fatal/Important 缺口，预计提升整体分 ≥ 8 分。
- 中影响：作用于 Important/Bonus 缺口，预计提升 3-8 分。
- 低影响：仅作用于 Bonus，预计提升 < 3 分。

### 1.2 可行性判定

- 高可行性：24h-1 周内可改完，且只动简历文本（补充量化、调顺序、补 bullet）。
- 中可行性：1-4 周，需新增 side project / 突击技能 / 重写项目细节。
- 低可行性：> 1 月，需补学历 / 考证 / 换业务线 / 转方向。

---

## 2. P0（高影响 × 高可行）— 24h 内可改

### 2.1 典型 P0 项

- 补充量化数据：项目描述无任何数字，但用户能回忆出（耗时下降、QPS、用户数、Bug 率）。
- 调整关键词对齐：简历用"前端"，JD 用"Web 开发/前端工程"，补齐 JD 高频技能词的同义表达。
- 重排技能区顺序：把 JD 必备技能前置，把无关技能下压。
- 补充 STAR 结果句：仅有"做了什么"，缺"结果如何"，用户能即时补。
- 修正明显错位：把隐藏在"其他"区的相关项目提到项目区前列。

### 2.2 P0 建议措辞模板

```
[P0] {动作} → {预期提分}
问题：简历第 {N} 段「{段落标题}」{现状描述}。
改法：{具体改写，给出可直接替换的句子草稿}。
依据：JD 第 {M} 条要求「{JD 原文}」。
预期：{维度}维度 +{X} 分，整体分 +{Y} 分。
```

### 2.3 P0 硬规则

- P0 必须给出"可直接替换的句子草稿"，不能只说"请补充量化数据"。
- 单次报告 P0 项不超过 5 条，避免用户被淹没；按预期提分降序取前 5。

---

## 3. P1（高影响 × 中可行）— 1 周内可改

### 3.1 典型 P1 项

- 重写 STAR bullet：现有 bullet 流水账，需按"情境-任务-行动-结果"重构（用户提供素材，AI 重写）。
- 补充项目细节：项目描述过简，需补技术决策、难点、方案选型。
- 必备技能突击佐证：JD 必备技能简历无项目佐证，需在 1-2 周内做一个最小 Demo 并写入项目区。
- 软技能事例化：把"沟通能力强"改为带具体跨部门协作事例的 bullet。

### 3.2 P1 建议措辞模板

```
[P1] {动作} → {预期提分}
问题：简历第 {N} 段「{段落标题}」{现状描述}。
改法：{1 周内可完成的具体动作 + 验收标准}。
依据：JD 第 {M} 条要求「{JD 原文}」，当前为 Important 缺口 G{k}。
预期：{维度}维度 +{X} 分，整体分 +{Y} 分。
```

---

## 4. P2（中影响 × 高可行）— 可快速优化

### 4.1 典型 P2 项

- 调整技能排列顺序（与 P0 重排区分：P2 是次要技能微调）。
- 补充 Bonus 技能的轻量佐证（如 GitHub 链接、技术博客）。
- 统一术语口径（如 JD 用"微服务"，简历写"服务化"，统一为 JD 用词）。
- 删减无关经历篇幅，把版面让给相关经历。

### 4.2 P2 建议措辞模板

```
[P2] {动作} → {预期提分}
改法：{快速优化动作}。
预期：{维度}维度 +{X} 分（影响较小，不阻塞投递）。
```

---

## 5. P3（低影响 / 低可行）— 长期提升

### 5.1 典型 P3 项

- 补学历（Fatal 学历缺口）。
- 考必备证书（Fatal 证书缺口）。
- 积累年限（Fatal 年限缺口，不可压缩）。
- 转业务方向（Important 领域不匹配且不可迁移）。
- 沉淀行业背景加分项（Bonus 行业缺口）。

### 5.2 P3 建议措辞模板

```
[P3] {动作} → 长期
说明：此为长期提升项，{预计周期}，不阻塞当前投递决策。
建议：{放弃该 JD / 中长期规划动作}。
注：时间不可造假，切勿通过虚构年限/学历来"补齐"。
```

---

## 6. 强制规则

1. **每条建议必须指向简历具体段落**：`target_paragraph` 字段不得为空，格式"简历第 N 段「段落标题」"。无法定位段落的建议不得输出。
2. **每条建议必须关联缺口**：`related_gap` 字段引用 `gap-classification.md` 输出的 gap id（G1/G2...）。无缺口支撑的建议视为无源建议，禁止输出。
3. **预期提分必须可累加验证**：所有 P0+P1 预期提分之和不得超过"100 - 当前整体分 + 15"（15 为重写带来的一致性溢价上限），超过则降级部分建议。
4. **不编造数据**：P0/P1 的"句子草稿"中涉及的具体数字必须来自用户提供的简历或追问，AI 不得自行填入未确认的数字（用 `{{待用户确认: 具体数值}}` 占位）。
5. **优先级排序输出**：报告中建议按 P0 → P1 → P2 → P3 分组，组内按预期提分降序。

---

## 7. 输出结构

```
improvements:
  - id: I1
    priority: P0 | P1 | P2 | P3
    action: "..."
    target_paragraph: "简历第 N 段「标题」"
    related_gap: G1                    # 引用 gap-classification 输出
    jd_reference: "JD 第 M 条「原文」"
    expected_gain:
      dimension: skill
      points: 8
    draft_sentence: "..."             # P0/P1 必填，P2/P3 可空
```
