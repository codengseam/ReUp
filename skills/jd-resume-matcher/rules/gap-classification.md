# gap-classification.md — 缺口分级规则

> 被 `../SKILL.md` 第 7 步与 `matching-model.md` 联动调用。本文件把"JD 要求 vs 简历差距"切为三级，每级含识别规则、影响程度、修复难度、措辞模板。AI 必须为每个识别出的缺口标注分级。

## 0. 分级总览

| 级别 | 含义 | 对整体分影响 | 处置 |
|------|------|------------|------|
| Fatal（致命） | 硬性条件不满足 | 封顶 45 或淘汰 | 直接放弃/转方向，不改简历 |
| Important（重要） | 核心技能缺失或经验不足 | 单维度扣 15-30 分 | 需补强，进 P0/P1 改进 |
| Bonus（加分） | 非核心技能/软技能欠缺 | 单维度扣 5-15 分 | 可选提升，不阻塞投递 |

---

## 1. Fatal（致命缺口）

### 1.1 识别规则

满足任一即判 Fatal：

- 学历低于 JD 下限一档（如 JD 硕士、简历本科）。
- 工作年限低于 JD 下限，且 JD 无"优秀可放宽"措辞；或差距 > 1 年即使有放宽措辞。
- JD 标注"必备"的证书缺失（CPA / 法考 / 执业医师 / 教师资格等准入类）。
- 简历存在造假信号（见 `boundaries.md` §造假识别），一旦识别即整体判 Fatal 并终止评分。

### 1.2 影响程度

- 单项 Fatal → 整体分封顶 45，报告置顶告警。
- ≥ 2 项 Fatal 或学历低两档 / 年限差距 > 3 年 → 整体分 = 0，建议放弃。

### 1.3 修复难度

- 学历缺口：长期（1-3 年），P3 级。
- 年限缺口：不可压缩（时间不可造假），P3 级。
- 必备证书缺口：中期（3-12 月备考），P3 级。

### 1.4 措辞模板

```
【刚性不达标】JD 要求 {刚性项} 为「{JD 要求}」，您的简历为「{简历实际}」。
此为致命缺口，整体匹配分已封顶至 45（或判定为不推荐投递）。
建议：{放弃该 JD / 转向 {替代方向} / 通过 {学历提升|证书考取} 中长期补强}。
不做建议：不要通过虚构学历/年限/证书来"凑齐"硬性条件。
```

## 2. Important（重要缺口）

### 2.1 识别规则

满足任一即判 Important：

- JD"必备技能"中，简历完全未体现或仅停留在"了解"层级（自评且无项目佐证）。
- 经验领域不匹配：JD 要求"电商支付"，简历全为"内部工具"，领域相关性 < 40%。
- 量化产出缺失：JD 强调"数据驱动/结果导向"，简历项目描述全部无任何量化指标。
- 管理岗管理经验不足：JD 要求"带 10 人团队"，简历无管理经历或仅带 2-3 人。
- 年限差距 ≤ 1 年且 JD 含"优秀可放宽"（从 Fatal 降级而来）。

### 2.2 影响程度

- 单项 Important → 所属维度扣 15-30 分（视缺失程度）。
- 累计 ≥ 3 项 Important → 整体分额外扣 10 分（避免短板叠加被低估）。

### 2.3 修复难度

- 必备技能缺失：中期（1-3 月突击 + 项目实战），P1 级。
- 经验领域不匹配：长期（需换业务线），P3 级；若可提炼"可迁移能力"则降为 P1。
- 量化产出缺失：短期（24h 内补充），P0 级。
- 管理经验不足：不可短期补，P2-P3；可用"项目主导/虚拟团队"部分替代，转 P1。

### 2.4 措辞模板

```
【重要缺口】{维度}：JD 要求「{JD 要求描述}」，您的简历「{简历现状描述}」。
影响：{对匹配度的具体影响，如"技能维度预计扣 25 分"}。
建议（{优先级}）：{可执行动作，如"在 XX 项目中补充 React Hooks 实战描述，并量化组件复用率"}。
指向段落：简历第 {N} 段「{段落标题}」。
```

## 3. Bonus（加分缺口）

### 3.1 识别规则

满足任一即判 Bonus：

- JD"加分技能"（标注"优先/加分/plus"）缺失。
- JD 列出的非必备软技能（如"有开源贡献优先"）简历未体现。
- 行业背景加分项缺失（如 JD"金融背景优先"，简历无金融经历但核心技能匹配）。
- 工具/方法论加分项缺失（如"熟悉 Design System 加分"）。

### 3.2 影响程度

- 单项 Bonus → 所属维度扣 5-15 分。
- 不触发额外累计扣分，不阻塞投递。

### 3.3 修复难度

- 加分技能缺失：短期（1-4 周 Demo 项目），P2 级。
- 软技能加分缺失：短期（补充佐证事例），P0-P2 级。
- 行业背景：中期，P3 级（不建议为加分项转行）。

### 3.4 措辞模板

```
【加分缺口】{维度}：JD 提及「{加分项}」为加分项，您的简历暂未体现。
影响：较小（{维度}维度扣 {5-15} 分），不影响投递。
建议（{优先级}）：{可选动作，如"若有相关 side project 可补充至项目区"或"暂不建议为此项专项投入"}。
```

## 4. 缺口与改进优先级的映射

缺口级别 ≠ 改进优先级。映射规则（详细矩阵见 `improvement-priorities.md`）：

| 缺口级别 | 默认优先级 | 例外 |
|---------|----------|------|
| Fatal | P3（长期/不可改） | 造假信号 → 立即停止，不进优先级 |
| Important | P0 或 P1 | 量化产出缺失 → P0；技能缺失 → P1；领域不匹配 → P3 |
| Bonus | P2 | 有现成 side project → P0 |
