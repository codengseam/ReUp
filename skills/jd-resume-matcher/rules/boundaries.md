# boundaries.md — 边界与失败模式

> 被 `../SKILL.md` Boundary 节调用。AI 在输出报告前必须读取本文件并逐项自检。任一失败模式命中须在报告中明示降级或拒绝信号，不得隐瞒。

## 1. 何时不信任匹配分

匹配分是"基于可见文本的结构化推断"，以下情形匹配分不可信，须标注 `confidence_degraded` 并降级：

| 情形 | 信号 | 处置 |
|------|------|------|
| 简历造假 | 项目时间重叠异常、公司名拼写近似知名公司、量化数字与岗位常识严重不符（如实习生"主导亿级 DAU 系统"） | 整体判 Fatal，终止评分，报告标注"造假信号，匹配分不输出" |
| 简历信息严重不足 | 简历 < 200 字或仅有岗位名/公司名，无职责描述 | 暂停，要求用户补充简历，不得基于贫信息打分 |
| JD 模糊 | JD < 100 字或仅有"招前端，待遇面议"类 | 暂停，要求用户补充 JD，不得自行脑补 JD 要求 |
| JD 内部矛盾 | 同一 JD 既要求"3 年经验"又要求"应届生"，或薪资与年限严重错位 | 标注 `jd_inconsistent`，按更严格的那条刚性校验，并在报告中提示矛盾 |
| 跨文化/跨国 JD 未标注 | JD 含 visa/语言要求但简历无对应信息 | 软技能维度标 `insufficient_info`，不计 0 分 |

## 2. 不做的事

### 2.1 不做"成功画像建模"

本 skill **不做**基于历史录用数据的"成功候选人画像预测"。原因：无历史录用数据集，无法训练或推断"什么样的人会被这家公司录用"。任何"根据大数据您被录用概率 X%"的表述均禁止。

### 2.2 匹配分 ≠ 录用概率

匹配分衡量的是"简历能力与 JD 要求的 D-A 互补程度"，不等于"录用概率"。录用还受以下本 skill 不覆盖因素影响：

- 候选人供给量（同岗位竞争者强弱）
- 用人部门预算与 headcount
- 面试表现（软技能现场表达）
- 薪资匹配（N-S 方向）
- 内部推荐/关系
- 时机与运气

报告中必须含一句声明：「匹配分反映岗位胜任度，不等于录用概率」。

### 2.3 不做 P-O Fit 评分

本 skill 只做 P-J Fit 的 D-A 方向，**不做**企业文化/价值观匹配（P-O Fit）评分。若 JD 含明显文化信号，仅在报告中单列"文化适配提示"供人参考，不计入匹配分。

### 2.4 不做过度资历惩罚

当 Abilities 远超 Demands（如 P8 资历投 P5 岗），本 skill 当前版本不主动降分。原因：缺历史数据标定"过度资历-离职风险"曲线。AI 应在报告中提示"过度匹配风险"供人决策，但不改分。

### 2.5 不替用户做投递决策

本 skill 输出诊断与建议，不输出"投/不投"的二元指令。最终投递决策由用户做出，AI 仅给出分数段语义建议（见 `matching-model.md` §3.3）。

## 3. 与 ATS 分的区别

| 维度 | 本 skill 匹配分 | ATS（Applicant Tracking System）分 |
|------|---------------|----------------------------------|
| 消费者 | 人（候选人）+ AI 后续消费 | 机器（HR 初筛系统） |
| 评分对象 | 简历文本能力对齐 | 简历关键词命中 + 字段可解析性 |
| 是否考虑语义 | 是（VSM 余弦相似度） | 多数 ATS 否（字面关键词 + 同义词词典） |
| 是否给改进建议 | 是（P0-P3 段落级） | 否（ATS 只过滤不给建议） |
| 是否考虑刚性 | 是（学历/年限/证书闸门） | 是（ATS 硬筛） |
| 分数含义 | 岗位胜任度 | 通过初筛的概率 |

报告中应提示：「本匹配分关注能力对齐与改进，与 ATS 系统的关键词过滤分不是同一指标；通过本 skill 优化后仍需确保简历字段对 ATS 可解析（如日期格式、技能词标准化）」。

> ATS 七类匹配维度（Skills / Job Titles / Education / Certifications / Management Level / Industries / Languages）参考 `../references/matcher-refs.md` 中 resumeoptimizerpro.com 条目。

## 4. 置信度规则

每份报告必须输出 `confidence`（0-1），按下表合成：

| 因子 | 取值 | 影响 |
|------|------|------|
| 简历信息完整度 | 完整 1.0 / 有空窗或模糊段 0.85 / 严重不足 0.5 | 信息越少越低 |
| JD 信息完整度 | 完整 1.0 / 模糊 0.6 / 严重模糊 0.3 | 同上 |
| 刚性层是否触发造假信号 | 否 1.0 / 是 0.0 | 触发即 0，终止 |
| 弹性层语义相似度是否触发 overreach | 否 1.0 / 是 0.7 | 降权但继续 |
| 是否有可考证量化数据 | 多 1.0 / 少 0.85 / 无 0.7 | 软技能维度尤其敏感 |

```
confidence = 简历完整度 × JD 完整度 × 造假因子 × overreach因子 × 量化因子
```

`confidence < 0.5` 时报告必须置顶「置信度低，结论仅供参考」。

## 5. 失败模式自检清单（输出前必跑）

AI 在生成最终报告前，逐项确认：

- [ ] 简历与 JD 均非空且信息充分，否则已暂停索取
- [ ] 未触发造假信号；若触发，已标注并终止评分
- [ ] 刚性层已逐项校验，Fatal 封顶规则已正确应用
- [ ] 弹性层语义相似度 < 0.3 的维度已限速
- [ ] 权重表已按岗位类型选取，动态调整已归一化并明示
- [ ] 每条改进建议均指向简历具体段落且关联缺口 id
- [ ] P0/P1 句子草稿中无 AI 自行编造的数字
- [ ] 报告含"匹配分 ≠ 录用概率"声明
- [ ] confidence 已计算，< 0.5 已置顶告警
- [ ] 未输出"录用概率 X%"类表述

任一项未通过，不得输出报告，返回对应前置步骤修正。
