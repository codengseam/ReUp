# 胜任力对齐规则（执行细则）

> 本文件基于 McClelland (1973) 胜任力理论与 Spencer & Spencer (1993) 冰山模型，定义简历中胜任力识别与对齐的执行规则。AI 必须按本规则做显性/隐性胜任力分层。

## 1. 理论基础

### 1.1 McClelland 胜任力理论
1973 年 McClelland 在《Testing for Competence Rather than Intelligence》中提出：传统的智力测验（IQ/学历）不能有效预测工作绩效，应以"胜任力"（competency）替代——即能区分高绩效者与普通绩效者的可测量个人特征。

核心论点：
- 学历/智力是"必要非充分"条件，胜任力才是绩效差异的根因。
- 胜任力需通过行为证据（critical incident）识别，而非自陈问卷。
- 不同岗位的胜任力模型不同，需基于岗位定制。

### 1.2 Spencer 冰山模型
Spencer & Spencer (1993) 在《Competence at Work》中将胜任力分为水面以上的"显性"与水面以下的"隐性"两层，共 6 类：

| 层级 | 胜任力类别 | 可见性 | 培养难度 | 简历中的体现 |
|------|-----------|--------|---------|-------------|
| 水上 | 1. 知识 (Knowledge) | 显性 | 易 | 学历、证书、技术名词 |
| 水上 | 2. 技能 (Skill) | 显性 | 中 | 经历中的工具使用、动作动词 |
| 水下 | 3. 社会角色 (Social Role) | 隐性 | 较难 | 团队协作、跨部门推动描述 |
| 水下 | 4. 自我认知 (Self-Image) | 隐性 | 难 | "我认为/我坚信"价值观表述 |
| 水下 | 5. 特质 (Traits) | 隐性 | 很难 | 抗压、严谨、主动的行为证据 |
| 水下 | 6. 动机 (Motives) | 隐性 | 极难 | 主动承担、超越本职的驱动描述 |

冰山模型要点：水上知识与技能易识别易培养；水下四层难识别但决定长期绩效，是高潜力候选人的关键区分点。

## 2. 胜任力分类与简历识别

### 2.1 显性胜任力识别（知识 + 技能）
显性胜任力直接从简历文本提取，无需推断：
- **知识**：教育背景的学历/专业/课程、持有的认证证书、技术名词的广度。
- **技能**：技能栏的工具列表、经历中动作动词后的技术栈、可观察的具体操作。

识别方法：关键词匹配 + 经历上下文验证（与 `rules/diagnostics-8dim.md` 维度 6 联动）。

### 2.2 隐性胜任力识别（社会角色/自我认知/特质/动机）
隐性胜任力需从行为证据推断，不能仅凭自陈：
- **社会角色**：跨团队协作、Owner 意识、对外代表的描述。证据如"代表团队与 X 部门对齐"、"作为 Owner 推动落地"。
- **自我认知**：对自身定位与价值观的表述。证据如"坚持质量优先"、"认为技术应服务业务"。
- **特质**：行为模式体现的稳定特征。证据如"在紧迫 deadline 下交付"（抗压）、"主动发现并修复历史遗留问题"（严谨/主动）。
- **动机**：驱动行为的内在需求。证据如"主动承担无 KPI 的重构"（成就动机）、"花业余时间带新人"（亲和/利他动机）。

> 规则：隐性胜任力必须有"行为证据"支撑，禁止仅凭"我很有责任心"等自陈形容词判定。无行为证据则标注"隐性胜任力证据不足"。

## 3. 对齐方法

### 3.1 显性 vs 隐性胜任力识别流程
```
1. 从 JD/岗位模型抽取目标胜任力清单（显性 + 隐性）。
2. 遍历简历，逐条提取行为证据，标注其体现的胜任力类别。
3. 显性胜任力：直接比对技能/知识是否覆盖。
4. 隐性胜任力：从行为证据推断，标注证据强度（强/中/弱）。
5. 输出对齐矩阵：{胜任力项 → 是否覆盖 + 证据 + 强度}。
```

### 3.2 胜任力对齐矩阵模板

| 目标胜任力 | 类别 | 简历是否覆盖 | 证据（原文片段） | 证据强度 | 缺口建议 |
|-----------|------|------------|----------------|---------|---------|
| 分布式系统设计 | 技能（显性） | 是 | "主导微服务拆分..." | 强 | - |
| 跨团队协作 | 社会角色（隐性） | 部分 | "与算法团队对接" | 弱 | 补充推动落地的具体动作 |
| 成就动机 | 动机（隐性） | 否 | - | - | 补充主动承担的非 KFA 项目 |

证据强度判定：
- **强**：有完整 STAR 行为证据 + 量化结果。
- **中**：有行为描述但无量化或场景模糊。
- **弱**：仅一笔带过，无具体行为。
- **无**：完全缺失。

## 4. 评分维度

### 4.1 胜任力覆盖度评分（0-10）
| 分数 | 标准 |
|------|------|
| 0-2 | 仅覆盖显性知识/技能，隐性四层完全无证据 |
| 3-4 | 显性覆盖较全，隐性仅 1 层有弱证据 |
| 5-6 | 显性完整 + 隐性 2 层有中等证据 |
| 7-8 | 显性 + 隐性 3 层有强证据 |
| 9-10 | 六层均有行为证据，且隐性层有稀缺亮点 |

### 4.2 胜任力-经历匹配度评分（0-10）
评估简历声称的胜任力是否与经历证据匹配（防止注水）：
| 分数 | 标准 |
|------|------|
| 0-2 | 声称的胜任力（如"精通分布式"）< 20% 有经历证据 |
| 3-4 | < 50% 有证据 |
| 5-6 | 50%-70% 有证据 |
| 7-8 | > 70% 有证据，且强度匹配 |
| 9-10 | 几乎全部有证据 + 隐性胜任力证据充分 |

> 本评分与 `rules/diagnostics-8dim.md` 维度 6（技能-经历一致性）互补：维度 6 聚焦技能词层面，本评分扩展至隐性胜任力层面。

## 5. 对齐输出格式

```
【胜任力对齐报告】
- 显性胜任力覆盖：{知识项 N/M} + {技能项 N/M}
- 隐性胜任力覆盖：
  - 社会角色：{覆盖/部分/无} — 证据：{原文}
  - 自我认知：{覆盖/部分/无} — 证据：{原文}
  - 特质：{覆盖/部分/无} — 证据：{原文}
  - 动机：{覆盖/部分/无} — 证据：{原文}
- 关键缺口：{列出未覆盖且 JD 看重的胜任力}
- 建议补充：{针对缺口的具体经历改写方向}
```

## 6. 与子 skill 的协作

- 显性胜任力识别由本规则直接执行。
- 隐性胜任力的"动机/特质"层证据不足时，触发编排 `competency-model-alignment` 子 skill——其"经验-技能-潜力-动机"四层模型专门用于挖掘潜力与动机证据。
- 胜任力缺口若涉及"未做过的盲区"，触发编排 `blind-spot-navigation` 子 skill 给出坦诚降维表述。
