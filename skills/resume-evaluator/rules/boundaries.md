# 边界与失败模式

> 本文件定义 `resume-evaluator` skill 的适用边界、失败模式与邻近方法论区分。AI 在执行前必须先核对本边界。

## 1. 何时不用此 skill

### 1.1 JD-简历深度匹配分析
- 场景：用户的核心诉求是"我的简历与这份 JD 匹配度多高、差在哪、怎么针对性改"。
- 区分：本 skill 的 ATS 评分仅为可选增强项，不做 JD 维度（硬性要求/职责/重点项）的逐项匹配打分。
- 路由：走 resume-jd-matching 主入口（基于 `src/features/resume/matcher.ts` 的 `buildMatchReportFromJD`）。

### 1.2 纯排版/语法润色
- 场景：用户简历内容已定稿，只需调整字体/间距/模板美观度。
- 区分：本 skill 评估内容质量，不做视觉设计。
- 路由：建议用户使用排版工具或设计师。

### 1.3 简历造假包装
- 场景：用户要求"帮我编一段大厂经历"或夸大不存在的项目。
- 红线：本 skill 严禁编造经历、虚构量化数据、伪造职位。这是诚信红线，触发即拒绝并提示风险。
- 说明：`highlight-extractor` 是榨取真实果汁，不是无中生有兑水。

### 1.4 非求职场景的简历
- 场景：学术简历（科研/教职）、艺术作品集、移民用途简历。
- 区分：本 skill 针对企业求职简历，学术/艺术简历有不同评价标准（如论文/作品权重）。

### 1.5 纯英文/海外岗位
- 场景：简历为纯英文，目标岗位在海外。
- 区分：本 skill 默认中文岗位，ATS 规则（关键词分词、职位头衔用语）需切换至英文区域模型。
- 路由：仍可使用本 skill，但需明确标注"英文岗位模式"，按 `rules/ats-scoring.md` §7.2 调整。

## 2. 作者警告的失败模式

### 2.1 量化注水
- 表现：为了让"内容量化度"维度拿高分，硬塞不真实或无意义的数字（如"提升了 1000%")。
- 后果：面试追问即穿帮，比没数字更糟。
- 防范：所有量化必须可溯源（用户能说清数字来源），无溯源的数字标注"待核实"。

### 2.2 关键词堆砌
- 表现：为了 ATS 覆盖率，在技能栏堆 50 个词或在总结段生硬塞关键词。
- 后果：ATS 可能过初筛，但人工 review 时显得不专业；且维度 6（技能-经历一致性）会判定注水。
- 防范：关键词必须自然嵌入经历上下文，孤立关键词不计入有效覆盖。

### 2.3 STAR 模板化
- 表现：所有 bullet 套用同一 STAR 句式，失去个性，像填表。
- 后果：HR 读 3 条就腻，差异化亮点维度（维度 8）扣分。
- 防范：STAR 是结构骨架不是字面模板，Action 段应体现技术决策的多样性。

### 2.4 隐性胜任力过度推断
- 表现：从一句"参与了团队协作"就推断出"强社会角色胜任力"。
- 后果：胜任力对齐报告失真，误导候选人。
- 防范：隐性胜任力必须有"强/中"强度行为证据，弱证据标注"证据不足"而非覆盖。

### 2.5 把职责当成果
- 表现：诊断时把"负责订单系统"当作已完成的成果给高分。
- 后果：维度 2/3 评分虚高。
- 防范：职责描述不计入量化度与 STAR 质量评分，必须区分"做了什么"与"负责什么"。

## 3. 容易混淆的邻近方法论

### 3.1 与 resume-jd-matching 的区分
| 维度 | resume-evaluator（本 skill） | resume-jd-matching |
|------|------------------------------|-------------------|
| 依赖 JD | 可选（ATS 为增强项） | 必须 |
| 核心产出 | 8 维诊断 + STAR 重写 | 匹配度分 + 差距分析 + 优先级 |
| 评分侧重 | 简历本身质量 | 简历与 JD 的契合度 |
| 编排子 skill | highlight/competency/blind-spot | 不编排或仅调用本 skill |

### 3.2 与 highlight-extractor 的区分
- 本 skill：宏观诊断器，评估简历整体并打分。
- highlight-extractor：微观素材榨取工具，从单件平淡经历中榨取亮点。本 skill 在维度 8 得分低时编排它。

### 3.3 与 competency-model-alignment 的区分
- 本 skill：评估简历整体的胜任力覆盖（六层冰山）。
- competency-model-alignment：构建面试回答的四层递进结构（经验-技能-潜力-动机）。本 skill 在隐性胜任力证据不足时编排它做深度挖掘。

### 3.4 与 blind-spot-navigation 的区分
- 本 skill：识别简历中暴露的能力盲区（技能声明无证据）。
- blind-spot-navigation：面试中面对"不会答的硬伤"的战术补救。本 skill 在识别盲区后编排它给出表述话术。

### 3.5 STAR 法则 vs 本 skill 的 8 维
- STAR 法则：单条 bullet 的叙事结构标准，是维度 3 的判定依据。
- 8 维诊断：简历整体的多元评估框架，STAR 只是其中一维。不可用 STAR 单维替代 8 维综合诊断。

## 4. 执行前的边界检查清单

AI 在开始诊断前必须核对：
- [ ] 用户诉求是"诊断简历整体"而非"匹配特定 JD"？（若后者，转 resume-jd-matching）
- [ ] 用户是否要求造假/夸大？（若是，拒绝并提示红线）
- [ ] 简历是否为图片化/扫描件无法解析文本？（若是，提示改用可解析格式）
- [ ] 简历语言与目标岗位是否匹配本 skill 默认（中文）？（若英文，切英文模式）
- [ ] 简历是否为学术/艺术等非企业求职场景？（若是，提示本 skill 适用性有限）
