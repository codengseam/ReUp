# 8 维诊断执行清单

> 本清单是 `rules/diagnostics-8dim.md` 的勾选式执行版。AI 诊断时逐项核对，每维 3-5 个检查点，全部勾选后据实打分。

## 维度 1：结构完整性

- [ ] 基本信息（姓名 + 至少 2 种联系方式 + 求职意向）齐备
- [ ] 工作经历模块存在且含公司/角色/时间/bullet
- [ ] 项目经历模块存在（与工作经历分离）
- [ ] 技能模块存在且有分类
- [ ] 教育背景模块存在
- [ ] 模块顺序符合惯例（基本信息 → 工作 → 项目 → 技能 → 教育）

**打分依据**：缺失模块数 / 层次混乱程度 → 0-10

---

## 维度 2：内容量化度

- [ ] 统计工作+项目 bullet 总数 N = ___
- [ ] 统计含数字（%/万/倍/QPS/DAU/ms 等）的 bullet 数 M = ___
- [ ] 量化率 = M/N = ___%
- [ ] 抽查 3 个量化 bullet 是否含对比基线（"从 X 到 Y"）
- [ ] 是否存在"大量/若干/部分"等不可衡量词

**打分依据**：量化率分档（<30% → 3-4；30-60% → 5-6；60-85% → 7-8；>85% → 9-10）

---

## 维度 3：STAR 叙事质量

- [ ] 抽查 5 个 bullet，标注各含 S/T/A/R 哪些要素
- [ ] bullet 是否以强/中动作动词开头（非"负责/参与/协助"）
- [ ] Action 段是否具体到工具/技术（如"用 Redis"而非"做了缓存"）
- [ ] Result 段是否独立可识别
- [ ] 是否存在"我们团队"泛泛而谈（无"我"的个人贡献）

**打分依据**：完整 STAR 的 bullet 占比 + 动词层级 → 0-10

---

## 维度 4：时间线连续性

- [ ] 工作经历按时间排序，标注每段起止年月
- [ ] 检查是否有 > 3 个月空窗（记录：___ 至 ___，共 ___ 个月）
- [ ] 检查是否有时间重叠（同期两段全职且无兼职说明）
- [ ] 时间格式是否统一（YYYY.MM / YYYY 年 MM 月 / 英文月份不混用）
- [ ] 最近一段经历是否持续至今或近 2 年内结束

**打分依据**：空窗长度 + 重叠 + 格式统一度 → 0-10

---

## 维度 5：关键词密度

- [ ] 若有 JD：提取 JD Top-20 关键词，计算覆盖率 = ___%
- [ ] 若无 JD：基于岗位类型通用高频词表，覆盖率 = ___%
- [ ] 关键词是否仅出现在技能栏（孤立关键词）→ 记录孤立词数 ___
- [ ] 是否有中英文同义词都覆盖（如 K8s + Kubernetes）
- [ ] 工作经历中是否自然嵌入核心关键词

**打分依据**：覆盖率分档 + 是否孤立（ATS ≥75% 对应 ≥7 分）

---

## 维度 6：技能-经历一致性

- [ ] 提取技能栏技能总数 = ___
- [ ] 在经历中检索每个技能词，有证据的技能数 = ___
- [ ] 一致率 = 有证据/总数 = ___%
- [ ] 检查"精通"声明的技能是否有强证据（架构/调优/排障场景）
- [ ] 是否存在熟练度虚高（全部标"精通"）

**打分依据**：一致率分档（<20% → 0-2；<50% → 3-4；50-70% → 5-6；>70% → 7-8）

---

## 维度 7：错别字/格式

- [ ] 逐行扫描错别字，记录数量 = ___（含同音字/英文拼写）
- [ ] 检查中英文标点混用（中文句号用"."等）
- [ ] 检查中英文之间是否有空格
- [ ] bullet 长度是否均衡（15-40 字为宜，记录最长/最短）
- [ ] 是否有图片化文本/复杂表格/特殊符号（ATS 不可解析）

**打分依据**：错别字数分档（≥5 → 0-2；3-4 → 3-4；1-2 → 5-6；0 → 7+）

---

## 维度 8：差异化亮点

- [ ] 识别"非日常"描述（主导/0-1/架构/跨团队/开源/专利）数量 = ___
- [ ] 评估亮点稀缺性（多数同级候选人能否写出？）
- [ ] 亮点是否有量化结果支撑
- [ ] 亮点是否与目标岗位相关
- [ ] 是否有加分项（个人总结/技术博客/开源项目/GitHub）

**打分依据**：稀缺亮点数 + 量化支撑（<5 分触发编排 highlight-extractor）

---

## 完成后交叉校验

- [ ] 维度 5 与维度 6 互校（关键词密度高但一致性低 → 两维均降）
- [ ] 维度 2 与维度 3 互校（量化高但 STAR 低 → 维度 3 降）
- [ ] 维度 7 否决项（错别字 ≥5 → 整体置信度降一档）
- [ ] 8 维分数全部填入 `scripts/scoring-formula.md` 计算总分
