# 提问安全区三元模型

## 概念 (Concept)

### I — 方法论骨架 (Interpretation)

面试尾声的反问环节是展示"成熟职场人"心态的关键时刻。
很多应聘者要么问"薪水多少/加不加班"（纯个人利益视角，显得功利），要么问"公司未来的战略是什么"（宏观且虚伪，显得讨好）。
真正高质量的提问，必须落在三个圆的交集区：
1. **应聘者关心的**（如工作强度、成长空间）。
2. **面试官关心的**（如你能不能干活、能不能解决他的麻烦）。
3. **职位客观要求的**（如项目的阶段、面临的具体挑战）。
在这个交集区内提问，既能刺探到你想知道的真实信息（如加班情况），又能向面试官展现出你"已经准备好解决问题"的专业素养。

## 解释 (Explanation)

### E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **收集用户真实诉求**
   - 问用户：你在面试这家公司时，内心最想知道的、最担忧的 1-2 个点是什么？（比如：怕背锅、怕没前途、怕天天加班）。
   - 完成标准: 获得用户的核心痛点。
   - **[检查点]**: 必须暂停，等待用户回答痛点后再进入下一步。

2. **定位三元交集**
   - 分析该痛点如何转化为面试官和职位也关心的角度。
   - **转化对照表**：
     | 用户痛点 | 三元交集转化 | 示例问题 |
     |---------|-------------|---------|
     | 怕加班 | 团队交付挑战 | "团队目前面临的最大技术挑战是什么？" |
     | 怕没成长 | 新人产出期望 | "这个岗位前半年最重要的产出目标是什么？" |
     | 怕背锅 | 项目质量保障 | "团队如何做代码评审和上线流程？" |
     | 怕技术栈落后 | 技术演进规划 | "团队未来一年有技术升级的计划吗？" |
   - 完成标准: 为每个痛点生成一个映射视角的概念。

3. **设计反问话术**
   - 用职场化的语言，将概念包装成 2-3 个具体的、开放式的问题。
   - **高质量反问模板**：
     ```
     【技术向】"请问团队目前面临的最大技术挑战是什么？如果我有幸加入，您希望我能在哪些方面快速贡献？"
     【业务向】"这个项目目前的用户规模和增长情况如何？下一步的产品重点是什么？"
     【团队向】"团队的代码评审和技术分享机制是怎样的？新人一般多久能独立承担模块？"
     【成长向】"您认为在这个岗位上做得最出色的人，他身上有哪些特质？"
     ```
   - 完成标准: 输出一组带有解释的高质量反问列表。
   - **[检查点]**: 向用户展示反问列表，确认是否符合其面试风格，暂停等待反馈。

4. **提供避坑指南**
   - 提醒用户面试尾声的"绝不解释失误规则"。
   - **常见错误及规避**：
     - ❌ 错误："刚才那个问题我没答好，其实我还想补充..." → ✅ 正确：直接跳过，用反问展示专业度
     - ❌ 错误："你们公司加班多吗？有加班费吗？" → ✅ 正确：问交付周期和团队挑战
     - ❌ 错误："公司未来五年的战略是什么？" → ✅ 正确：问团队未来一年的技术规划
   - 完成标准: 明确告知用户不要在反问环节试图解释之前没答好的问题。
   - **[检查点]**: 确认用户理解避坑要点，暂停等待确认。

### B — 边界 (Boundary) ★

#### 不要在以下情况使用此 skill

- 面试的开场或技术考核中途，此时应该专注回答问题，而不是反问面试官。
- 面对 HR 面试时（HR 更关注通用素质和薪酬框架，对技术团队的具体交付挑战不了解，此时应直接谈薪或问企业文化，不适用复杂的三元业务模型）。

#### 作者在书中警告的失败模式

- **绝不解释失误规则**：在提问环节，不要为了挽回前面的表现而发表"虽然我不会，但我会努力学"的表白。
- **过早暴露底牌的谈薪僵局**：在价值尚未被面试官充分认可的初面，不要急着问薪资福利，这会让后续谈薪陷入被动。

#### 容易混淆的邻近方法论

- 面试套话大全（单纯给问题清单，而不教背后如何与面试官需求交集，效果往往不好）。

## 原文 (Original Text)

### R — 原文 (Reading)

> "应聘者、面试官、职位本身，这三者的需求内容是有交叉的...应聘者的问题，最好是与职位相关的，并且是应聘者、面试官都关注的信息。作为应聘者，如果你仅仅从自己需要的角度提问...面试官会觉得你只关注个人成长...而如果你只问面试官和职位需要的信息...又显得有些讨好。"
>
> — 白海飞, 《面试现场》25讲

## 场景 (Scenario)

### A1 — 书中的应用 (Past Application)

#### 案例 1: 探听团队的真实工作强度
- **问题**: 候选人非常想知道这个岗位是不是常年 996，但直接问"你们加班多吗"会引起反感。
- **方法论的使用**: 利用三元模型，将问题包装成对团队现状的关注。
- **结论**: 问出"请问目前团队面临的最大挑战是什么？我们这个项目的常规交付周期是怎样的？"
- **结果**: 面试官不仅没有觉得被冒犯，反而详细描述了项目的高压状态和赶工期，候选人从而准确评估了背后的高强度加班文化，同时给面试官留下了"关注项目交付"的好印象。

### A2 — 触发场景 (Future Trigger) ★

#### 用户会在什么情境下需要这个 skill?

1. 用户马上要去面试，不知道最后"你有什么想问我的"该准备什么问题。
2. 用户想打听薪资福利、团队氛围、晋升空间等敏感话题，怕问得不好。
3. 用户在准备面试提纲，觉得网上的反问模板都太千篇一律了。

#### 语言信号 (用户的话里出现这些就应激活)

- "面试官最后问我有什么问题，我该问啥？"
- "我想知道他们公司加不加班，怎么委婉地问？"
- "帮忙准备几个高质量的反问问题。"
- "怎么打听薪水和试用期比较合适？"

#### 与相邻 skill 的区分

- 与 `blind-spot-navigation` 的区别: 盲区降维是用来**防守**面试官的硬核提问；而本技能是用来在面试末尾**主动出击**，刺探公司情报。
