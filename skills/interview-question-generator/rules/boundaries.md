# boundaries.md — 边界与失败模式

> 被 `../SKILL.md` Boundary 节调用。AI 在输出题集前必须读取本文件并逐项自检。任一失败模式命中须在题集中明示降级或拒出信号，不得隐瞒。
> 题型分类见 `question-taxonomy.md`；难度分级见 `difficulty-grading.md`；答案引导模板见 `answer-guidance-template.md`；RAG 外挂见 `rag-hook-spec.md`。

## 1. 何时不出题（信息不足阈值）

题集质量上限由输入信息量决定。以下情形须判停或降级，不得在贫信息上强行出题：

| 情形 | 信号 | 处置 |
|------|------|------|
| 简历信息严重不足 | 简历 < 200 字，或仅有姓名/岗位/公司，无项目段落、无技术栈描述 | 判停，向用户索取更完整简历，不出题 |
| JD 信息严重不足 | JD < 100 字，或仅有"招前端，待遇面议"类，无职责/技术栈/职级 | 判停，向用户索取更完整 JD，不出题 |
| JD 缺职级信号 | JD 无"高级/资深/带团队/X 年经验"等任一职级线索，且 `jinsheng-san-yuanze` 无法推断 band | 降级：难度定级回退到 L1+L2 为主（默认 P5/P6 档），不出 L3/L4，题集置顶标注"职级未明，难度按 P5/P6 默认档" |
| 简历与 JD 技术栈无交集 | 简历技术栈与 JD 要求技能零重叠（如简历纯前端、JD 要求 Java 后端） | 判停并提示"简历与 JD 技术栈不匹配，技术深挖/八股文题无法基于简历生成"，建议用户先核对岗位 |
| 简历造假信号 | 项目时间严重重叠、公司名拼写近似知名大厂、量化数字与岗位常识严重不符（如实习生"主导亿级 DAU 系统"） | 判停，标注"简历含造假信号，不出题"，建议用户核实经历真实性 |
| JD 内部矛盾 | 同一 JD 既要求"应届"又要求"5 年经验"，或薪资与年限严重错位 | 标注 `jd_inconsistent`，按更严格的那条校验，并在题集置顶提示矛盾 |

判停与降级的区别：判停 = 完全不出题，返回索取信息提示；降级 = 仍出题但缩范围并标注。

## 2. 避免造题（基于实际内容，禁止编造）

**核心红线**：每道题必须可溯源到简历或 JD 的实际段落，标注 `resume_ref` 或 `jd_ref`。禁止凭空编造简历上没有的技术栈、项目、成果。

| 禁止行为 | 反例 | 正确做法 |
|---------|------|---------|
| 编造技术栈 | 简历未提 Kafka，却出"你用过 Kafka 吗？讲讲 ISR 机制" | 只出简历/JD 交集技术栈的题；JD 要求但简历未提的，归入 `blind-spot-navigation` 盲区降维预案，不作为"你用过"题 |
| 编造项目 | 简历无该项目，却追问"你那个 XX 系统的 QPS 是多少" | 项目追问题只针对简历中实际出现的项目段落 |
| 编造成果 | 简历写"优化了性能"，题目预设"你优化后 QPS 提升到 10 万" | 追问"优化前后用什么指标衡量、提升了多少"，让候选人自证，不预设数字 |
| 把团队成果当个人成果 | 题目默认候选人做了全部工作 | 必含"个人贡献边界"追问（你做了哪部分，同事做了哪部分），见 `question-taxonomy.md` §2 |
| 把 JD 假设当简历事实 | JD 要求"高并发经验"，题目问"你在简历里的高并发项目…"而简历并无高并发项目 | JD 要求但简历无对应经历的，归入缺口防守（`blind-spot-navigation`），不出"你做过"题 |

**溯源标注规则**：
- 技术深挖题、项目追问题：必须标 `resume_ref`（简历段落引用）。
- 系统设计题、行为面试题：必须标 `jd_ref`（JD 要求段落引用），若有简历关联经历可叠加 `resume_ref`。
- 八股文题：标 `source`（题库来源或 `verified: false` 标记），见 `rag-hook-spec.md`。
- 任一题目缺少溯源标注，视为造题嫌疑，须回退补标或删除。

## 3. 不出歧视性 / 违规题（红线清单）

以下属性属于受保护属性，**任何题型、任何难度均不得**作为题目内容、考察点或追问方向。命中即删除该题并记录：

- 性别、性别认同与性取向
- 年龄、出生年份（"你今年多大了""打算工作到几岁"）
- 户籍、地域、籍贯（"你是哪里人""会回老家吗"）
- 婚育状况（"结婚了吗""打算要孩子吗""孩子谁带"）
- 民族、种族
- 宗教信仰
- 健康状况、残障、遗传病史
- 政治面貌（**例外**：JD 明确要求党员身份的岗位，如部分国企/机关，可就"党员身份匹配"做事实确认，不得做价值观评判）
- 家庭背景、家庭成员职业、家庭经济状况
- 既往薪酬具体数字（可作为候选人主动反问，不作为面试官追问题）

**间接歧视陷阱**：题目虽不直接提及受保护属性，但答案会诱导候选人暴露该属性时同样禁止。例如"你父母支持你来这个城市吗"（诱导家庭/地域）、"你打算在这边长期发展吗"（诱导婚育/户籍意图）。

违规题不可通过"降低难度"或"改写措辞"洗白，须直接删除。

## 4. 与单纯刷题库的区别（诊断式出题，非题海）

| 维度 | 本 skill（诊断式出题） | 单纯刷题库 |
|------|----------------------|-----------|
| 出题依据 | 简历实际内容 + JD 实际要求 + 能力缺口，三者交集 | 通用技术考点，与个人无关 |
| 题目定制度 | 每题溯源到简历/JD 段落，题目内容含候选人真实项目/技术栈 | 通用题干，任何人拿到都是同一题 |
| 题集结构 | 6 类题型按配比 + L1-L4 难度梯度，复现三层递进 | 扁平罗列，无配比与梯度设计 |
| 配套引导 | 每题配 STAR 回答引导 + 关键追问 + 评分要点 + 常见错误 | 多数仅给参考答案，无引导与追问 |
| 缺口处理 | 含 `blind-spot-navigation` 盲区降维预案 | 不处理个人缺口 |
| 消费者 | AI（RAG 检索 + skills-loader 注入）+ 人 | 人为主 |

**判定规则**：用户意图若是"给我出 N 道 XX 技术的面试题练手"，且不提供简历/JD，则**不路由到本 skill**，直接走 RAG 八股文题库直查（见 `rag-hook-spec.md` fallback 或 `src/server/rag/search.ts`）。本 skill 仅在用户提供简历 + JD（或至少其一 + 缺口列表）时激活，产出定制题集。

报告/题集中须含一句声明：「本题集基于个人简历与 JD 定制生成，非通用题海；纯刷八股文请直接检索题库」。

## 5. 与 reverse-questioning-framework 的协作边界

反向提问题由本 skill 编排 `reverse-questioning-framework` 生成，二者职责分工如下：

| 职责 | 归属 |
|------|------|
| 决定"是否需要反问题""出几个" | 本 skill（按 `question-taxonomy.md` §5 配比，通常 1-2 题，反问总数 2-3 个） |
| 反问内容的三元交集建模（应聘者需求 ∩ 面试官需求 ∩ 职位要求） | `reverse-questioning-framework` 内部逻辑，本 skill 不重写、不覆盖 |
| 反问的 `jd_ref` 溯源标注 | 本 skill（在编排结果上补标） |
| 反问落在三元交集哪个区的标注 | `reverse-questioning-framework` 产出，本 skill 透传不修改 |

**边界**：
- 本 skill **不**自创反问话术，所有反问文本须来自 `reverse-questioning-framework` 的编排输出，避免两套逻辑打架。
- 当用户**只**要反问准备（"帮我准备几个面试反问"）时，直接路由到 `reverse-questioning-framework`，**不**走本 skill 的 6 题型编排流程（见 `question-taxonomy.md` 题型选择决策表）。
- 反向提问题不参与 L1-L4 难度定级（反问无难度梯度，按 `question-taxonomy.md` §5 仅标"反向提问"），不占用 L4 配额。

## 6. 置信度规则

每份题集须输出 `confidence`（0-1），按下表合成：

| 因子 | 取值 | 影响 |
|------|------|------|
| 简历信息完整度 | 完整（有项目+技术栈+成果量化）1.0 / 有空窗或模糊段 0.85 / 严重不足 0.5 | 信息越少越低，< 0.5 判停 |
| JD 信息完整度 | 完整 1.0 / 模糊 0.6 / 严重模糊 0.3 | 同上 |
| 职级是否可推断 | 可推断 band 1.0 / 不可推断（降级 P5/P6 档）0.8 | 影响难度定级置信度 |
| 八股文题库命中率 | 全命中 1.0 / 部分命中 0.85 / 全 fallback（AI 生成）0.7 | fallback 题须标 `verified: false` |
| 是否触发造假信号 | 否 1.0 / 是 0.0 | 触发即 0，判停 |

```
confidence = 简历完整度 × JD 完整度 × 职级因子 × 题库命中因子 × 造假因子
```

`confidence < 0.5` 时题集必须置顶「置信度低，题集仅供参考，建议补充输入信息后重新生成」。

## 7. 输出前自检清单（输出前必跑）

AI 在输出题集前，逐项确认：

- [ ] 简历与 JD 均非空且信息充分，否则已判停索取，未在贫信息上强行出题
- [ ] 每道题均标注 `resume_ref` 或 `jd_ref` 或 `source`，可溯源到输入实际段落
- [ ] 无编造技术栈/项目/成果数字；JD 要求但简历无的经历已归入 `blind-spot-navigation`，未作为"你做过"题
- [ ] 无歧视性/违规题（性别/年龄/地域/婚育/民族/宗教/健康等受保护属性均未出现，含间接歧视陷阱）
- [ ] 题型配比符合 `question-taxonomy.md` 总览表（技术岗偏 1/2/6，管理岗偏 3/4/5）
- [ ] 难度梯度覆盖 L1 到目标职级对应级别，无全 L1 或全 L3
- [ ] L4 题不超过 1 道，且仅 P8+ 目标才出（见 `difficulty-grading.md`）
- [ ] 八股文题来自 RAG 题库或已标 `verified: false`（见 `rag-hook-spec.md`）
- [ ] 反向提问题来自 `reverse-questioning-framework` 编排，本 skill 未自创反问话术
- [ ] 每题含完整答案引导（参考答案 + STAR 引导 + 关键追问 + 评分要点 + 考察点 + 难度 + 常见错误），见 `answer-guidance-template.md`
- [ ] `confidence` 已计算，< 0.5 已置顶告警
- [ ] 题集含"本题集为定制诊断题集，非通用题海"声明

任一项未通过，不得输出题集，返回对应前置步骤修正。
