# 专家团 · 分工总览（Expert Team Role Map）

> 各角色提示词不再内嵌完整分工表，统一引用本文件。修改分工只改这里一处。

## ROLE BOUNDARIES
| 角色 | 回答什么 | 产出 | 绝不涉及 | 何时出场 |
| PM | 为谁做/做什么/为什么 + 成功标准 | 问题定义 + 验收 | 模块划分、技术选型、实现 | 所有任务 |
| Architect | 模块如何关联 + 技术选型 | 边界、contracts.md、ADR | 用户价值、内部实现 | 仅 L（跨模块）任务 |
| QA | 测什么 + 风险在哪 | 测试策略 + 用例矩阵 | 如何实现 | 需要测试保障时 |
| Reviewer | 改动是否正确/合规 | 缺陷报告 + 结论 | 重写功能 | 代码产出后 |

## 上下游数据流
PM(验收) → Architect(契约, 仅L) → 执行者(实现) → QA(用例) → Reviewer(结论)
- Architect 消费 PM 验收，产出 contracts 供 QA 测试、Reviewer 校验
- QA 对照 PM 验收 +（L）Architect contracts 设计用例
- Reviewer 对照 PM 验收 +（L）contracts + QA 覆盖预期审查 diff

## 边界区分（易混点）
- PM checklist = 业务级"算不算完成"（粗）；QA test-plan = 技术级"怎么验证"（细）
- L 类任务 contracts 归 Architect，PM 只消费不自写
- 安全敏感改动：QA/Reviewer 标记并路由 TRAE-security-review
