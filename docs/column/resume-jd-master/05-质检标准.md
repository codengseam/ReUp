# 05 — 质检标准：给 AI 与人共同参照的验收基线

> 本篇是「简历-JD 分析专栏」的第五篇，定义专栏与 4 个 skill 包的质检标准，给 AI 与人共同参照。它不教方法论（方法论见 [01-简历分析规则](./01-简历分析规则.md)、[02-JD分析规则](./02-JD分析规则.md)、[03-组合分析规则](./03-组合分析规则.md)、[04-面试题生成规则](./04-面试题生成规则.md)），只定义"怎样算合格、怎样算不合格"。

## 一、为什么需要统一的质检标准

专栏与 skill 包是"AI 消费优先"的产物：SKILL.md 是 AI 入口，rules 是 AI 执行的硬约束，prompts 是结构化提示词。AI 消费物与传统文档最大的区别在于——一处占位符、一处内部矛盾、一处不可溯源的论断，都会被模型当真并放大到每一次推理输出里。所以质检不能靠"读一遍觉得没问题"，必须有可机器化、可逐项勾选的清单。

本篇把质检拆为四个角色视角：PM 验收（完整性）、QA 用例（行为正确性）、Reviewer 缺陷扫描（文本质量）、反幻觉检查（可溯源性），再用一个专家团 loop 把四者串成闭环。

## 二、PM 验收清单（完整性）

PM 关心"该有的有没有"。五项逐条勾选，任一未通过则不予验收：

- [ ] **4 个 skill 包结构完整**：每个 skill 包含 SKILL.md + rules/ + scripts/ + references/ + prompts/ 五件套。SKILL.md 是薄路由层，不重复细则；细则下沉到 rules/；执行流与模板下沉到 scripts/；引用独立成文件；提示词全保留。
- [ ] **6 篇专栏文章完整**：00 序章 + 01 简历分析 + 02 JD 分析 + 03 组合分析 + 04 面试题生成 + 05 质检标准，每篇聚焦"可分析/建设的规则"，引用规范。
- [ ] **引用可溯源**：每条引用标注作者、年份、标题、来源、关键论点摘要；学术文献回指具体条目，行业实践表述为"参考某产品公开做法"而非"官方合作"。
- [ ] **编排链正确引用现有 skills**：4 个新 skill 包正确 composes-with / reuses 现有 8 个 skills，不重复造方法论，只做路由与加权聚合。
- [ ] **对接代码路径准确**：专栏中提到的对接代码路径（如 `src/lib/rag/`、matcher 实现路径）与实际代码库一致，不写不存在的文件名。

PM 验收的判定标准是"二值"：完整或不完整，不存在"基本完整"。缺一个文件即整包退回。

## 三、QA 用例矩阵（行为正确性）

QA 关心"按规则跑会不会出错"。用例分三层：

- **每篇专栏 ≥ 3 黄金用例**：输入 → 预期输出。例如 03 组合分析的黄金用例：给定一份本科 3 年经验的简历 + 一份要求硕士 5 年的 JD，预期触发学历 Fatal 封顶 45、年限 Fatal，整体分 = 0，报告置顶刚性不达标告警。黄金用例的作用是把专栏里的规则"钉死"成可验证的行为契约。
- **每个 skill ≥ 5 测试 prompt**：覆盖正常 / 边界 / 失败三类路径。正常路径验证主流程；边界路径验证阈值（如年限差距恰好 1 年、语义相似度恰好 0.3）；失败路径验证判停（如简历 < 200 字、JD 内部矛盾、造假信号）。
- **反幻觉检查**：所有论断可溯源到引用，引用可溯源到原始来源（详见第五节）。

QA 用例的产出物是 `skills/*/test-prompts.json` 与专栏配套的黄金用例集。用例不是写完就完，每次规则变更必须回归。

## 四、Reviewer 缺陷扫描（文本质量）

Reviewer 关心"文本本身有没有毛病"。四类缺陷零容忍：

- **无占位符**：全文不得出现 TBD / TODO / FIXME / `xxx` / 待填。规则要么写明，要么标注为"未来扩展方向"并说明当前不可用、AI 不得假装已获取——这是"诚实的空白"与"占位符"的区别。
- **无内部矛盾**：同一规则在不同文件中的表述一致。例如权重表在 `dimension-weights.md` 与专栏 03 中的数字必须一致；Fatal 封顶 45 在 `matching-model.md`、`gap-classification.md`、`boundaries.md` 三处的描述必须自洽，不能一处说 45 一处说 50。
- **无歧义表述**：每个术语首次出现时定义清楚，避免"匹配分"在不同语境下被理解为不同含义。专栏 03 专门区分了"匹配分"与"ATS 分"、"P-J Fit"与"P-O Fit"，就是消除歧义的示范。
- **范围聚焦无蔓延**：每个 skill 只做自己的事，不越界。jd-resume-matcher 不做 P-O Fit 评分、不做成功画像建模、不做过度资历惩罚、不替用户做投递决策——边界明确写入 `boundaries.md` 并在专栏中复述。

Reviewer 扫描的产出物是缺陷清单，每条缺陷标注文件、行号、缺陷类型、修复建议。

## 五、反幻觉检查规则（可溯源性）

反幻觉是 AI 消费物的生命线。规则有两条递进的链：

**第一条链：论断 → 引用**。专栏与 skill 中的每个论断（如"P-J Fit 与绩效相关 0.44-0.56"）必须可溯源到具体引用条目。学术文献的论点摘要为二次概括，AI 不得声称"直接引用原文"，应表述为"基于 Kristof-Brown 等 (2005) 元分析结论"。行业实践条目为公开产品/案例的二次整理，AI 不得声称"与某产品官方合作"，仅表述为"参考某产品公开做法"。

**第二条链：引用 → 原始来源**。每条引用必须可溯源到书名 + 作者 + 页码/链接。学术文献回指期刊、卷期、页码；行业实践回指产品文档 URL 或公开案例；书籍回指书名、作者、章节。无法溯源的引用不得写入——宁可少一条引用，不可编造一条。

两条链合起来即"双层可溯源"：论断指向引用，引用指向原始来源。这是 McClelland 1973 胜任力"可测量性"原则在质检上的延伸——胜任力须有可观测行为佐证，论断须有可溯源证据佐证；也是 Kristof-Brown 2005 元分析"可验证性"的延伸——元分析的价值在于效应量可复现，论断的价值在于可被独立核验。

反幻觉检查在 AI 推理时还有一条兜底：所有 skill 的 rules 中，凡涉及"不得"的硬规则（不得编造数字、不得出简历未提及的技术深挖题、不得声称录用概率、不得把 AI 现编答案冒充题库标准答案），AI 在输出前必须逐项自检。任一命中则不得输出，返回前置步骤修正。

## 六、专家团 loop 流程

四类质检不是各跑各的，而是串成一个闭环，对应专家团（Orchestrator / PM / Architect / QA / Reviewer）的分工：

```
PM 验收（完整性）→ QA 用例（行为正确性）→ Reviewer 缺陷扫描（文本质量）→ 反幻觉检查（可溯源性）→ 修复 → 复检
```

1. **PM 验收**：先过完整性闸门。结构不完整直接退回，不进入后续环节。
2. **QA 用例**：结构完整后跑黄金用例与测试 prompt。用例失败记录为行为缺陷。
3. **Reviewer 缺陷扫描**：跑文本质量扫描，记录占位符/矛盾/歧义/蔓延缺陷。
4. **反幻觉检查**：跑双层可溯源扫描，记录不可溯源论断。
5. **修复**：所有缺陷汇总，按严重程度排序修复。Fatal 级缺陷（占位符、内部矛盾、不可溯源论断）必须修复，Bonus 级缺陷（表述微调）可择优修复。
6. **复检**：修复后重跑 1-4 步，直到零 Fatal 缺陷。复检必须全量回归，不能只验修改项——防止修复引入新矛盾。

这个 loop 借鉴 PDCA（计划-执行-检查-行动）持续改进框架：验收与用例是"计划+执行"，三类扫描是"检查"，修复与复检是"行动"。loop 不是一次性流程，每次规则迭代都要重跑——专栏与 skill 是"活"的文档，质检也是"活"的流程。

## 七、质检通过判定

专栏与 skill 包通过质检的充要条件：

- PM 验收 5 项全勾选；
- QA 黄金用例与测试 prompt 全部通过；
- Reviewer 扫描零 Fatal 缺陷（占位符/矛盾/歧义/蔓延）；
- 反幻觉检查零不可溯源论断；
- 复检全量回归通过。

任一条件未满足，标注"待修复"状态，不得声明"已完成"。这与项目级规则一致——不得在未验证前声称完成。验证手段是 `pnpm ts-check && pnpm lint && pnpm test`（代码侧）与本篇的质检 loop（文档侧）并行执行。
