# 04 — 面试题生成规则：诊断式出题而非题海战术

> 本篇是「简历-JD 分析专栏」的第四篇，把 `skills/interview-question-generator/` 下的硬规则提炼为可读方法论。规则的权威来源是 skill 内 `rules/` 目录，本篇面向人讲清"出什么题、按什么难度出、每题配什么答案引导、边界在哪"。组合分析的缺口识别见 [03-组合分析规则](./03-组合分析规则.md)，整体质检见 [05-质检标准](./05-质检标准.md)。

## 一、诊断式出题 vs 题海战术

先厘清一个根本区别：本专栏的面试题生成**不是刷题库**。

题海战术的逻辑是"这套题每个人都做一遍"，题目与候选人个人经历无关，只与技术栈相关。它的价值在于扫描知识面宽度，但无法验证候选人简历里那些技术的真实深度，也无法针对 JD 的实际要求做精准考察。纯刷八股文题库，用 RAG 直查即可，不必走本 skill 的编排。

诊断式出题的逻辑是：基于"简历实际内容 + JD 实际要求 + 能力缺口"三者交集，按题型分类、按难度分级、配以答案引导，产出结构化题集。每道题必须可溯源到简历或 JD 的实际段落，禁止凭空编造简历上没有的技术栈或项目。题目之间不是扁平罗列，而是有逻辑关系的分层递进——这是白海飞《面试现场》考核逻辑的核心：面试官的问题分三层递进，表层事实（扫描经验面）→ 深度细节（探究能力高低与潜力）→ 感受和观点（考查动机与价值观）。生成的题集必须复现这种分层结构。

一句话：本 skill 是"基于个人简历 + JD 的定制题集"，不是通用题海。

## 二、六类题型方法论

题集由六类题型按配比组成，单次题集约 15-20 题。每类题型有明确的触发条件与考察目标。

### 1. 技术深挖题（20%-25%）

基于简历写明的技术栈，深挖原理、源码理解或踩坑经验，验证技术的真实深度。

触发条件是"简历技术名词"与"JD 必备技能"的交集——两者交集即为深挖靶点。每个靶点技术生成 1-2 题，按"原理 → 源码 → 踩坑"递进，必须可溯源到简历具体段落（标注 `resume_ref`）。**禁止出简历未提及的技术深挖题**——这是反幻觉的硬规则。

题干复现白海飞三层递进：表层（用过吗、核心机制）→ 深度（底层怎么实现、踩过最深的坑）→ 观点（重新选型你会怎么选、为什么）。考察的是技能深度、学习能力与技术选型判断力。

### 2. 项目追问题（20%-25%）

基于简历项目经历，用 STAR 反向追问细节，验证项目真实性与个人贡献度。

触发条件是简历含项目段落且 JD 关注交付能力。每个重点项目生成 2-3 个追问点，覆盖 STAR 四要素。这里有一条关键防线：**必须追问"个人贡献边界"**（你做了哪部分，同事做了哪部分），防止团队成果冒充个人成果。这是面试官识别简历注水的常用手段，也是题集必须复现的。

追问模板按 STAR 展开：背景与角色 → 具体动作与技术障碍 → 结果指标与重做改进 → 贡献边界。考察经验真实性、个人贡献度、复杂问题解决能力与复盘能力。

### 3. 系统设计题（15%-20%）

基于 JD 中高并发、大规模、高可用等要求，设计系统方案，验证架构能力。

触发条件是 JD 出现"高并发""千万级用户""高可用""分布式""性能优化"等关键词，且目标职级 P6+（P6 开始要求独立当一面，P7+ 要求跨团队架构）。先用 `jinsheng-san-yuanze` 的价值原则筛选 JD 中值得设计的高价值方向——能解决业务瓶颈的才算值得出题，不为出题而出题。

题目给出业务场景 + 约束条件（QPS、数据量、一致性要求），要求候选人给出架构方案。**必须包含权衡点**（如 CAP 取舍、成本 vs 性能），不出只有唯一标准答案的题——系统设计考的是思考过程与决策框架，不是背诵。初级岗 P5 可降为 0-1 题。

### 4. 行为面试题（15%-20%）

基于 JD 软性要求 + 简历经历，设计行为面试题，考查素质模型下层（潜力与动机）。

触发条件是 JD 出现"团队协作""带人""跨部门沟通""抗压""ownership"等软性要求，目标职级 P6+（P7+ 强制要求带团队/跨团队协作）。题目用"请举一个具体例子"开头，强制 STAR 回答，禁止表态式回答——"我觉得沟通很重要"不算回答，"上次我和测试在提测标准上产生分歧，最后我..."才算。

方法论根基是 BEI（行为事件访谈）：通过对过去行为具体细节的深度挖掘预测未来表现。这是 Spencer & Spencer 1993《Competence at Work》系统化的方法。行为题对应协作、领导、抗压、价值观与动机四类考察点。管理岗可上调配比至 25%。

### 5. 反向提问题（5%-10%）

候选人向面试官提问的环节，编排 `reverse-questioning-framework` 生成高质量反问。

触发条件是题集需覆盖面试全流程，或用户明确想准备"你有什么想问我的"环节。直接调用三元交集模型（应聘者需求 ∩ 面试官需求 ∩ 职位要求），每个反问标注其落在三元交集的哪个区。反问通常 2-3 个即可，考察职业成熟度、对职位的真实兴趣与信息刺探能力。

### 6. 八股文题（15%-20%）

基础知识题，RAG 外挂题库检索，验证技术基本功。

触发条件是 JD 技术栈有标准基础知识考点（如 TCP、HTTP、JVM、数据库索引、并发原语）且题库已有该技术栈的题目。难度以 L1-L2 为主——深度题归入技术深挖题，八股文只管基础概念。纯技术岗可上调配比，管理岗下调。

### 题型选择决策

输入信号决定优先题型：简历技术栈丰富且 JD 要求同栈 → 技术深挖 + 八股文；简历有重点项目 → 项目追问；JD 含高并发/大规模 → 系统设计；JD 含协作/领导/抗压 → 行为面试；用户要准备全流程 → 全部 6 类按配比；用户只要反问准备 → 仅反向提问；简历与 JD 有能力缺口 → 追加 `blind-spot-navigation` 盲区降维预案。

技术岗偏重 1/2/6，管理岗偏重 3/4/5。总量上限受 token 预算约束。

## 三、难度分级 L1-L4 与职级映射

难度对标**目标职级**（JD 要求的职级），而非候选人当前职级。题集应覆盖从 L1 到目标级别的完整梯度——让候选人既能答出基础题建立信心，又在高级题上暴露真实能力边界。

| 等级 | 名称 | 核心特征 | 对应职级 | 单级题量 |
|------|------|---------|---------|---------|
| L1 | 基础 | 概念/定义/记忆 | P5 | 1-2 题 |
| L2 | 进阶 | 应用/对比/分析 | P6 | 1-2 题 |
| L3 | 深度 | 设计/优化/权衡 | P7 | 1-2 题 |
| L4 | 极致 | 压力/反直觉/边界 | P8+ | 0-1 题 |

- **L1 基础**：考查记忆与复述，答案有明确标准，"读取大脑存储器"类问题。典型句式"简述 X 的定义/作用"。主要用于八股文与技术深挖表层。
- **L2 进阶**：考查场景应用、技术对比、问题定位，需要逻辑推导，答案不唯一但有明确评判维度。典型句式"在 XX 场景下为什么选 A 而不是 B"。这是 P6 区分"会用"与"理解"的核心题。
- **L3 深度**：考查系统设计、性能优化、技术权衡，需综合多个知识点给出方案并论证取舍，没有标准答案，考思考过程与决策框架。对应《大厂晋升指南》Why 类问题，是 P7 级别的核心区分题，占问题总数 50%-80%，级别越高占比越高。
- **L4 极致**：在压力情境下考查，或出反直觉题、边界条件题，故意挑战候选人已给出的答案验证深度与抗压。典型如"你刚才说 X 优于 Y，但如果条件变成 ZZ，结论还成立吗？"L4 对候选人压力极大，题集中最多 1 题，且仅 P8+ 目标才出，P5-P7 不出 L4。

职级-难度-题型映射总表：P5（L1 + 少量 L2，不出 L4，8-12 题）；P6（L1 + L2 + 少量 L3，不出 L4，12-15 题）；P7（L2 + L3 + 少量 L1，不出 L4，15-18 题）；P8+（L3 + L4 + 少量 L2，0-1 道 L4，15-20 题）。映射依据是李运华《大厂晋升指南》职级体系：P5 独立执行 / P6 独当一面 / P7 带小团队 / P8 领域影响力。

## 四、答案 + 引导模板

每道题不是只给题干，而是配完整的答案引导结构，供候选人自评与面试官对照：

```
题目 / 参考答案 / 回答引导 / 考察点 / 难度等级 / 常见错误回答
```

- **参考答案**：给出方向性答案，不是唯一标准（系统设计/行为题尤其如此）。
- **回答引导**：按 STAR + 关键追问 + 评分要点组织，告诉候选人"答到什么程度算合格、什么程度算优秀"。
- **考察点**：明确本题考的是能力模型的哪一层，便于针对性准备。
- **难度等级**：写入 `difficulty` 字段，必须与 L1-L4 映射自洽。
- **常见错误回答**：列出候选人的典型翻车点（如表态式回答、团队成果冒充个人成果、背诵八股却答不出原理），这是反幻觉与反套路的兜底。

不同题型有模板变体：技术深挖题的引导侧重"原理 → 源码 → 踩坑"三层递进；项目追问题侧重 STAR 四要素 + 贡献边界；行为面试题强制 STAR 且禁止表态；八股文题侧重标准答案 + 易错点。完整模板见 skill 内 `rules/answer-guidance-template.md`。

## 五、八股文 RAG 外挂接口设计理念

八股文题不是 AI 现编的，而是走 RAG 外挂接口检索题库。接口契约：输入（技术栈 + 难度 + 数量）→ 输出（题目 + 答案 + 标签）。

为什么预留外挂接口而不是让 LLM 直接生成？三个理由：

1. **可验证性**：题库里的题经过校对，答案可信；LLM 现编的八股文答案可能有细节错误（如 TCP 握手状态机记混），基础题一旦答错会误导候选人。
2. **覆盖度**：题库按技术栈系统化沉淀，LLM 现编容易扎堆在热门考点，漏掉冷门但常考的基础题。
3. **可演进**：题库可独立扩充，不需要改 skill 逻辑——新考点加入题库即可被检索到。

**fallback 策略**诚实而克制：题库未命中时，AI 可以生成并标记 `verified: false`（未经题库验证），明确告诉候选人"此题答案未经题库校对，请交叉验证"。绝不假装题库里有、绝不把 AI 现编的答案冒充题库标准答案。这与本专栏整体的反幻觉立场一致——详见 [05-质检标准](./05-质检标准.md) 的反幻觉检查规则。

接口与现有 `src/lib/rag/` 检索链路的对接方式见 skill 内 `rules/rag-hook-spec.md`，题库集成与检索调优见 `scripts/question-bank-integration.md`。

## 六、边界与红线

- **不造题**：每题必须可溯源到简历或 JD 的实际段落，禁止凭空编造简历上没有的技术栈或项目。
- **信息不足不出题**：简历/JD 缺关键字段（技术栈、项目细节、职级要求）时判停，向用户索取，不臆测不补全。
- **不出歧视性/违规题**：不涉及婚育、户籍、宗教、健康等受保护属性。
- **L4 克制**：L4 题最多 1 道，且仅 P8+ 目标才出，避免压垮候选人信心。
- **难度梯度完整**：题集必须覆盖从 L1 到目标级别的完整梯度，避免全 L1 或全 L3。

本 skill 是薄路由层，本身不承载全部细则，而是路由到 `rules/` 与 `scripts/`，并编排 `highlight-extractor`、`competency-model-alignment`、`jinsheng-san-yuanze`、`reverse-questioning-framework`、`blind-spot-navigation` 等现有 skills 协同产出。它解决的是"基于个人上下文定制题集"的编排问题，不重复造方法论。
