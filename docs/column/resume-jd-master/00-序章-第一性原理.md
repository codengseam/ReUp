# 序章：第一性原理

> 这是"简历-JD 分析专栏"的开篇。本专栏不是 AI 执行清单的复述，而是把背后"可分析、可建设的规则"提炼成给从业者看的方法论。同样这份内容也会被 RAG 检索消费——人读与机读同源。

## 一、第一性原理：三个因子的乘积

任何一份由 AI 产出的简历/JD 分析报告，其质量都可以用一个乘式近似：

```
AI 产出质量 = 提示词约束力 × 外挂知识密度 × 执行框架结构化度
```

之所以是乘积而非加和，是因为三者相互制约：任何一个因子趋近于零，整体就趋近于零。补强其中一个因子，并不能弥补另外两个的缺失。这也是过去同类工具"用起来总差点意思"的根因。

### 根因诊断：三者皆弱的现状

把市面上常见的"AI 帮我看简历""AI 解析 JD"实现代入这个公式，几乎都能命中同一种病灶：

- **提示词约束力弱**：系统提示只写了"你是一位资深 HR，请分析这份简历"。角色泛化、输出格式开放、评分标准缺失。模型只能凭训练语料的"平均印象"作答，于是产出四平八稳、放之四海皆准，既不指出具体扣分点，也不给出可执行的改写。
- **外挂知识密度低**：没有把胜任力理论、ATS 匹配机制、行业职级体系这些"领域事实"注入上下文。模型不知道"精通/熟悉/了解"对应不同能力境界，不知道 ATS 按 7 类加权而非字面计数，不知道"带领团队"和"管理团队"在职级推断上含义不同。密度低意味着模型在关键判断点上只能猜。
- **执行框架结构化度低**：没有规定"先拆字段、再建矩阵、再推断隐性、最后输出考察重点"的有序流程，而是让模型一次性吐出全文。结果是无法溯源——你不知道某个结论是从哪条 JD 原文推出的，也无法在出错时定位是哪一步崩了。

三者叠加，产出的就是"看着像那么回事、经不起追问"的分析。本专栏所对应的项目，正是用一组可执行的 skill 包把这三个因子逐一硬化：规则文件提供约束力，references 提供知识密度，scripts/execution-flow 提供结构化度。

## 二、三个功能的定位

围绕"简历"与"JD"两个对象，专栏归并出三类分析场景，各自有明确的产出边界：

| 场景 | 输入 | 核心产出 | 边界 |
|------|------|---------|------|
| 简历单独分析 | 一份简历（可选 JD） | 8 维诊断 + STAR 重写建议 + 胜任力对齐 | 不做 JD 维度逐项匹配；ATS 评分仅在提供 JD 时作为增强项 |
| JD 单独分析 | 一份 JD | 岗位画像（8 字段 + 能力矩阵 + 隐性要求 + 考察重点） | 不读简历、不做匹配打分、不预测"什么样的人会成功" |
| 组合分析 | 简历 + JD | 匹配度评分 + 缺口清单 + 优先级 + 面试题 | 拼合由匹配器完成，不在单侧 skill 内合并 |

这三类边界不是组织习惯，而是逻辑必然。单侧评估不构成"匹配"——P-J Fit 是双向概念，只看人或只看岗都不算 Fit，必须双侧对照。因此简历分析和 JD 分析各自只产出"单侧的结构化画像"，组合分析才把两侧拼合。混淆边界是常见的失败模式：用 JD 侧的 skill 直接给匹配分，或用简历侧的 skill 顺手解读 JD，都会引入幻觉。

## 三、方法论谱系

专栏背后的方法论不是凭空发明，而是一条可溯源的演进链。理解这条链，才能理解为什么规则这样设计。

```
胜任力理论 (McClelland, 1973)
        │  提出"胜任力替代智力测验"，行为证据优于学历
        ▼
冰山模型 (Spencer & Spencer, 1993)
        │  显性(知识/技能)易培养，隐性(角色/认知/特质/动机)决定长期绩效
        ▼
P-E Fit 元分析 (Kristof-Brown et al., 2005)
        │  证实人-岗匹配与绩效/满意度/留任显著正相关；隐性匹配预测力更高
        ▼
ATS 7 类匹配模型 (atsgrade, 2025)
        │  把"匹配"工程化为可计算的加权覆盖率：Skills 30-65%、放置位置权重、≥75% 强匹配
        ▼
AI 招聘实践 (北森 / Moka / 智联×通义千问, 2020-2026)
   JD 结构化解析 + 岗位画像 + 隐性推断置信度 + 可解释可溯源匹配报告
```

这条链的每一环都在专栏里有对应落点：

- McClelland 与 Spencer 支撑了胜任力对齐的 6 层分层、显性 vs 隐性的识别规则。
- Kristof-Brown 与 Edwards 划定了"单侧不做 Fit 判断""禁止成功画像建模"的边界。
- atsgrade 的 7 类模型支撑了 ATS 评分的加权公式与 75% 阈值。
- 北森/Moka/智联的工程实践支撑了 JD 8 字段抽取、职级推断、隐性推断置信度标注。

值得强调的是证据等级的差异：学术文献（McClelland/Spencer/Boyatzis/Kristof-Brown/Edwards）是高证据等级，行业实践（atsgrade/北森/Moka）是中等证据等级，中文课程（白海飞《面试现场》）是经验性证据。专栏在引用时会标注这一区别，避免把实践经验当成学术结论。

## 四、专栏结构与阅读指南

专栏正文按"总—分"组织，建议按序阅读：

- **00 序章（本篇）**：建立第一性原理与方法论坐标，理解三个因子的乘积与三类功能的边界。
- **01 简历分析规则**：从简历侧讲清 8 维诊断、ATS 评分、STAR 重写、胜任力对齐。适合"想诊断自己简历"或"想给候选人做简历 review"的读者。
- **02 JD 分析规则**：从 JD 侧讲清结构化拆解、能力矩阵、隐性推断、考察重点预测、职级推断。适合"想读懂一份 JD 到底要什么"的读者。
- **03 组合分析规则**：核心功能，讲清三层混合匹配模型（刚性校验 → 弹性语义匹配 → 多维加权）+ 缺口分级 + 改进建议优先级。适合"想做简历-JD 匹配诊断"的读者。
- **04 面试题生成规则**：讲清 6 类题型 + 难度分级 + RAG 八股外挂。适合"想针对岗位预生成面试题"的读者。
- **05 质检标准**：讲清 PM 验收 + QA 用例 + Reviewer 缺陷 + 反幻觉 + 专家团 loop。适合"想了解专栏质量保障机制"的读者。

01-05 各篇文末均有"本篇引用"段落，列出编号与简述，详细展开见 `_references/per-chapter/01-refs.md` 至 `05-refs.md`。专栏前端视图会隐藏 `_references` 目录，只展示正文，但引用关系始终保留以备溯源。

阅读时有一个提示：本专栏聚焦"规则与方法论"，不涉及代码实现细节。如果你需要知道某条规则对应哪个函数、哪个数据结构，请回到各 skill 包的 `rules/` 与 `scripts/` 目录——那里是给 AI 执行用的细则，本专栏是同一套规则的人类可读版提炼，两者同源但侧重不同。

## 本篇引用

本篇涉及的核心理论来源，详细展开见 [`_references/per-chapter/01-refs.md`](./_references/per-chapter/01-refs.md) 与 [`_references/per-chapter/02-refs.md`](./_references/per-chapter/02-refs.md)。

1. **McClelland (1973)** — 胜任力理论奠基，提出以行为证据替代智力测验，是显性/隐性二分法的理论源头。
2. **Spencer & Spencer (1993)** — 冰山模型六层结构，知识/技能在水上，社会角色/自我认知/特质/动机在水下，决定长期绩效。
3. **Kristof-Brown et al. (2005)** — P-J Fit 元分析，证实人-岗匹配与绩效/留任正相关，隐性匹配预测力高于显性，划定了单侧不做 Fit 判断的边界。
4. **Edwards (1991)** — P-J Fit 概念整合，强调 Fit 是双向概念，单侧评估不构成匹配。
5. **atsgrade (2025)** — ATS 7 类加权匹配模型，把"匹配"工程化为可计算覆盖率，支撑 75% 强匹配阈值。
