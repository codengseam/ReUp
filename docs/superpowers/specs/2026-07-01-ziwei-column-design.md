# 紫微斗数学习专栏设计规格 (v2 修订版)

> 日期：2026-07-01 ｜ 状态：待批准 ｜ 目标：学会紫微斗数，以之为镜认识自我不足并加以修改
> 修订依据：架构/内容/QA 三专家并行评估反馈整合

## 一、第一性原理分析

紫微斗数的本质：一套以出生时间(年月日时)为输入，通过天干地支、星曜、宫位、四化等符号系统构建命盘、推断人生轨迹的传统术数体系。其核心价值不在"算命"，而在提供一套认识自我性格特质、潜能倾向、人生节律的符号语言——这正是"当镜子"的基础。

要从零学会并用作修心之镜，需掌握五层能力：
1. 符号系统（星曜/宫位/四化含义）
2. 运算规则（排盘/大限/流年/小限）
3. 推断逻辑（格局/组合/三方四正/飞星）
4. 实战应用（解盘流程/定盘/各领域分析/合盘）
5. 心性转化（从认知到修养到改过行动）

由此推导出 6 大模块结构，每模块 6 篇，共 36 篇。

## 三、文章模板格式

每篇文章统一结构：
```
# 序号-标题

> 一句话导语

## 正文小节（按内容需要 4-7 节）
...

## 修心启示
（结合本篇具体星曜/宫位/四化内容，谈如何作为镜子认识自我、修改不足；
 须含至少1个可操作的自我观察或行为调整练习；修心篇则谈更深转化）

## 本篇小结
（3-5 条要点）

<!-- END_OF_DISPLAYED_CONTENT -->
## 参考资料
- 具体书目/课程/名家（须为主引用库子集）
```

格式约定：
- 简体中文，繁体术语首次出现括注
- 排盘示例用文字表格，不用图片
- 术语首次出现须引用术语表原文定义
- 交叉引用用相对路径链接如 `[第15篇](../03-四化飞星篇/15-宫干四化自化与飞化.md)`
- 防迷信化表述：禁用"命中注定/一定会/必然发生/改命"等宿命论措辞
- 星曜亮度表统一采用一份锁定版本（写入术语表）

## 四、引用资料处理（显式标记方案）

依用户要求，采用显式物理隔离 + 显式标记块双保险：
- **整体完整引用资料**：保留在 `ziwei-column/_references/master-references.md`，不展示
- **生成提示词**：保留在 `ziwei-column/_references/prompts.md`，不展示
- **单篇引用**：置于每篇文章 `<!-- END_OF_DISPLAYED_CONTENT -->` 标记之后的 `## 参考资料` 小节
- **渲染契约**：渲染器输出 `<!-- END_OF_DISPLAYED_CONTENT -->` 之前所有内容为专栏展示；`_references/` 整目录跳过
- **去重规则**：单篇参考资料须为主引用库子集，主库为权威超集

## 六、执行策略（专家团 + 共享契约 + 并行 + 3轮Loop）

### Phase 0：共享契约产出 + 专家团评估（已部分完成）
专家团评估已完成（架构/内容/QA三视角）。本阶段须产出5份共享契约文档作为6个写作agent的共同契约：
- `glossary.md`：术语表，每术语唯一权威定义（含36篇关键术语基线，见附录A）
- `style-guide.md`：标题层级上限、表格适用场景、每节字数区间、修心启示篇幅占比
- `school-baseline.md`：三合/中州/飞星/紫云/现代/象数宗核心主张、差异点、统一表述口径
- `cultivation-line.md`：每模块修心线主题 + 与正文结合的硬性要求
- `scope-boundaries.md`：每篇"讲什么/不讲什么"边界声明，防重复

### Phase 1-3：并行生成（每 Phase 2 模块 12 篇，2路并行）
- 每个写作 sub-agent 负责 1 个模块（6 篇），3 Phase 共 6 个写作 agent
- 每个 agent 收到：模块大纲、文章模板、5份共享契约、引用资料、验收标准
- 用 general_purpose_task subagent 执行

### Phase 4：专家团质检（并行4视角）
- 内容正确性质检：排盘数值/四化飞星方向/格局成立条件/术语跨篇一致性
- 结构完整性质检：36篇齐全、模块衔接、交叉引用准确
- 引用规范质检：END标记位置、单篇引用为主库子集、主库完整
- 修心镜像质检：修心启示含具体练习、防迷信化、"镜子→发现→修改"闭环

### Phase 5：3轮Loop修复（带收敛阈值）
- **Round 1**：修单篇内孤立问题（占位符、字数、小结条数、排盘数值错、术语定义缺失）— 按篇并行
- **Round 2**：修跨篇协调问题（术语一致性、流派表述、交叉引用、重复内容）— 按问题类别串行，同类禁止并行
- **Round 3**：全量一致性终检 + 残留扫尾 — QA串行复核
- **收敛阈值**：P0=0 且 P1≤3 为达标；Round 3 后 P0仍>0 触发规格回退
- **问题清单结构化**：每条含 issue_id/article_id/category/severity/location/description/suggested_fix/cross_article_refs/fix_round
