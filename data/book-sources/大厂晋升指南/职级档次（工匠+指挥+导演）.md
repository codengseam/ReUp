# 职级档次（工匠+指挥+导演）

上一讲我介绍了 COMD 能力模型，让你能够具体拆解一个级别的能力要求，不再纠结于抽象的描述。但你可能还是不清楚每个级别到底要求什么。这些具体要求，我会在课程第二部分，也就是职级详解部分一一介绍。

不过在这之前，我想先通过三个类比带你纵向透视职级档次，对不同档次的核心能力建立一个形象的认知。

## P5/P6：专业工匠

P5/P6 这一档相当于 “专业工匠”，就像木匠、铁匠、粉刷匠一样，核心能力是完成任务。

这里的任务是指每个岗位应当完成的事情，比如开发岗位应当完成代码编写，测试岗位应当完成测试用例执行。

P5 和 P6 的职责一样，比较简单，不应当太多解读。这两个级别的区别是，P5 应当在别人的指导下完成工作，而 P6 可以独立完成工作。其实只要有意愿在技术领域发展，基本上每个人都能达到 P6 的水平。

P5/P6 的核心职责如下表所示。

|岗位|P5/P6 核心职责|备注|
|---|---|---|
|开发|参与需求评审、编码、单元测试、方案评审、测试方案设计、测试用例执行、项目上线等事务|同上|
|测试|参与需求评审、测试用例设计、测试用例执行、方案评审、项目上线、线上运维等事务|同上|
|运维|参与需求评审、线上问题排查、系统部署、监控告警、方案评审、项目上线等事务|同上|
应当强调的是，这里列举的只是一个岗位的核心职责，并不代表这个岗位只做这些事情，比方说开发岗位的 高级工程师/专家/资深专家 也是要参与编码的。

另外，这里只列举了开发、测试和运维这些技术岗位的职责。产品、运营和市场等非技术岗位的同学，也可以根据你掌握的信息来整理你所在岗位的核心职责表格。

## 高级工程师/专家/资深专家：乐团指挥

高级工程师/专家/资深专家 这一档相当于 “乐团指挥”，核心能力是指挥团队。

为什么我要这么类比呢？由于 高级工程师/专家/资深专家 的职责和乐团指挥的职责非常相似。乐团指挥的核心工作职责，具体可以分为三个阶段：

1. **总谱研究**：对总谱进行深入细致的研究分析，识别和标注演奏的重点、难点和风险点。

2. **排练准备**：明确演奏应当的人手和乐器，根据乐团情况制定排练计划。

3. **正式排练**：拆解具体排练步骤（比如个体练习、分声部练习和全体排练等），抓好每一个关键环节的落实，做好风险预防措施，推动整个乐团完成演奏。

高级工程师/专家/资深专家 的任务和乐团指挥非常像，也可以分为三个阶段，跟乐团指挥的三个阶段正好一一对应。你只要把总谱换成团队的工作目标，把人手和乐器换成资源，把演奏排练换成工作目标落地就行了。

首先是**分析阶段**，对应乐团指挥的总谱研究；然后是**计划阶段**，对应排练准备；最后是**落地阶段**，对应正式排练。

那么 高级工程师/专家 和 资深专家 的区别是什么呢？高级工程师/专家 只应当指挥单个团队，而 资深专家 往往要指挥多个团队。

另外还应当补充一点，我这里说的 “团队”，包括两种类型：

1. **狭义上的团队**：组织结构上的行政级别团队，比如 高级工程师/专家 担任的 3 ~ 10 人团队的 Team Leader，负责团队管理、团队规划、团队考核和团队建设等管理职责。

2. **广义上的团队**：为了完成某个目标而成立的虚拟团队（或者说临时团队），比如某个项目投入的人员组成了 “项目团队”，某个专项的人员组成了 “专项团队”。

高级工程师/专家/资深专家 的核心职责如下表所示。

|岗位|高级工程师/专家 职责|资深专家 职责|
|---|---|---|
|开发|1. 负责单个系统的技术设计
2. 复杂项目的子系统方案设计
3. 单个系统的技术规划、技术演进
4. 带单个开发团队|1. 负责子领域里所有子系统的技术设计
2. 复杂项目的整体方案设计
3. 子领域的技术规划、技术演进
4. 带多个开发团队|
|测试|1. 单个系统的测试技术设计
2. 复杂项目的子系统测试方案
3. 单个系统的测试技术规划
4. 带单个测试团队|1. 子领域测试技术设计
2. 复杂项目的整体测试方案
3. 子领域测试技术规划、技术演进
4. 带多个测试团队|
|运维|1. 单个系统的运维技术设计
2. 设计运维优化方案
3. 单个系统的运维技术规划
4. 带单个运维团队|1. 子领域运维技术设计
2. 复杂项目子领域的运维方案
3. 子领域运维技术规划、技术演进
4. 带多个运维团队|
## 总监/P10：电影导演

总监/P10 这一档相当于 “电影导演”，核心能力是导演作品。

为什么我会这么类比呢，由于 总监/P10 的工作跟电影导演很像，具体表现在三个方面：

### 1. 工作规模

比如说你只是拍一段 60 秒的 vlog，还算不上电影导演；真正的电影导演拍出来的是几十分钟以上，剧本、服饰、化妆、道具、表演、运镜和剪辑都非常成熟的作品。同样地，P5 ~ 资深专家 这几个级别的工作都会产出一些成果，但这些成果在规模上还不足以跟 总监/P10 这个级别相比。

### 2. 总决策者

在一个剧组里，一般情况下导演就是老大，有绝对的话语权。同样地，虽然 P6 可以指导别人，高级工程师/专家/资深专家 可以带团队，但工作仍然会在很大程度上受到制约，关键的目标制定、资源整合和关键决策的工作，还是得由 总监/P10 来完成。

具体一点说，总监/P10 应当：

- 制定有挑战的业务目标；

- 整合不同的团队，包括多个技术团队（比如 Android、iOS、前端、Java 后端、测试、运维等）和多个业务团队（比如某头部大厂的广告平台的某个业务，可能涉及 QQ、微信和应用宝等多个业务团队）；

- 做出关键决策（例如要做什么、不做什么、先做什么和后做什么等）。

### 3. 总负责人

一部电影作品会打上导演的烙印，甚至呈现出强烈的导演个人风格。电影拍得不好，观众第一个骂的就是导演；拍得好，赞美和荣誉也首先给到导演身上。

同样地，总监/P10 的水平、眼界、价值观和做事风格，直接决定了一条业务线的质量，由于这些因素会体现在工作过程中的各种决策里面，决定了最终的呈现效果。

另外，导演往往有自己擅长的题材，比如文艺片、喜剧片；而 总监/P10 一般也都聚焦于某个业务或者专业领域，比如电商业务、出行业务、安全领域、算法领域，很少有跨领域样样精通的人才。

总监 和 P10 的核心差异在于成果质量。我还是拿电影导演来类比，总监 是成熟的导演，能拍出 7 分以上的作品（基本合格）；P10 是成名的导演，能拍出 8 分以上的作品（比较优质）。

虽然对于 总监/P10 的工作成果，并没有一个通用的打分机制，然而公司能通过一些硬指标来衡量，最典型的就是直接看业务结果。

如果你负责的业务结果实现了既定的业务目标，那么你就是成熟的导演，可以胜任 总监；如果你负责的业务结果按照某个标准（用户量、收入和权威机构的测评等），进入了业界前列，有一定的名气和影响力，那么你就是成名的导演，可以胜任 P10。

总监/P10 的核心职责如下表所示。

|岗位|总监 职责|P10 职责|
|---|---|---|
|开发|1. 业务整体的技术设计
2. 复杂项目的整体技术方案
3. 业务级的技术规划、技术演进
4. 带某个业务的多个或者所有技术团队|1. 业务整体的技术设计
2. 复杂项目的整体技术方案
3. 业务级的技术规划、技术演进
4. 带某个业务的所有技术团队，甚至更大的业务范畴|
|测试|1. 业务整体的测试技术设计
2. 复杂项目的整体测试方案
3. 业务级的测试技术规划、技术演进
4. 带某个业务的多个或者所有测试团队|同上，区别在于业务规模更大，影响力更大|
|运维|1. 业务整体的运维技术设计
2. 复杂项目的整体运维方案（例如机房规划、上云等）
3. 业务级的运维技术规划、技术演进
4. 带某个业务的多个或者所有运维团队|同上，区别在于业务规模更大，影响力更大|
## 这些类比有什么用

我用 “工匠、指挥、导演” 三种形象来类比不同的职级档次，然而这仅仅是针对职责的相似度来说的。**这种档次划分对应了行政级别的高低，但不代表艺术成就的高低**，比如一个顶级的木匠，艺术成就一定高于乐团指挥。

之因此要把职级档次做这样的划分，核心目的是为了让你快速抓住核心：当我们再说到某个级别的时候，你就能一下子抓住它的核心要点。

当然，这一讲的类比只是宏观层面的特征提炼。如果你想了解每个级别能力的细节，还是应当参考课程职级详解部分关于每个级别的详细解读。

由于某头部大厂的级别是业界的 “职级硬通货”，辨识度高，认可度高，因此我采用了某头部大厂 P5 ~ P10 的级别作为例子进行讲解。不管你在大公司还是小公司，不管你公司现在是否有完善的职级体系，如果你想了解自己能力水平在行业内所处的级别，我建议你都对标某头部大厂的职级来估计。

目前网络上已经有一些关于不同公司职级怎么对应的文章；而且我也专门准备了一期加餐，根据我的面试经验，提炼了几个典型互联网大厂的职级对应关系。这些信息你都可以参考。

## 本章小结

现在，我们回顾一下这一讲的重点。P5 ~ P10 的 6 个等级，可以根据能力特征分成 3 个档次，分别对应三种职业角色。

1. **P5/P6 相当于专业工匠**，核心能力是执行任务，P5 和 P6 的差别在于能否独立完成任务。

2. **高级工程师/专家/资深专家 相当于乐团指挥**，核心能力是指挥团队，高级工程师/专家 和 资深专家 的差别在于指挥的是单个团队还是多个团队。

3. **总监/P10 相当于电影导演**，核心能力是导演作品，总监 和 P10 的差别在于导演出来的是成熟的作品还是成名的作品。

我把这个对应关系总结在了下面这个表格里，供你参考。

|职级档次|对应角色|核心能力|级别差异|
|---|---|---|---|
|P5/P6|专业工匠|完成任务|P5：指导下完成；P6：独立完成|
|高级工程师/专家/资深专家|乐团指挥|指挥团队|高级工程师/专家：指挥单个团队；资深专家：指挥多个团队|
|总监/P10|电影导演|导演作品|总监：成熟作品；P10：成名作品|
最后再补充一点，**高级别的能力要求包含低级别的能力要求**。比如 总监 的核心能力是 “导演成熟作品”，但肯定也要具备 资深专家 要求的 “指挥多个团队” 的能力。
