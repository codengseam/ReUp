## 一、项目背景

### 1.1 业务背景

科大讯飞AI教育大数据综合服务平台是一个面向K12教育的智能化教育管理平台，利用人工智能和大数据技术，为教育主管部门、学校和教师提供全方位的教育教学解决方案。

**平台核心价值**：
- 教育数据采集与分析
- 智能化教学辅助决策
- 个性化学习路径推荐

**业务覆盖范围**：
- 覆盖40+区域教育局
- 服务50万+学子
- 支撑数百所学校的日常教学管理

### 1.2 核心应用模块

| 应用模块 | 功能说明 | 用户群体 |
|---------|---------|---------|
| 走班考勤 | 新高考改革下的走班制学生考勤管理 | 学校管理员、班主任 |
| 高考选科 | 新高考"3+3"/"3+1+2"选科指导与统计 | 学生、家长、学校 |
| 成绩分析 | 多维度成绩统计、对比、趋势分析 | 教师、学校管理者 |
| 智能排课 | 基于AI算法的自动排课系统 | 教务处 |
| 等级赋分 | 新高考等级赋分计算与转换 | 学校、教育局 |

## 二、从0到1搭建质量体系（STAR法则详解）

### 2.1 背景（Situation）

**入职初期面临的问题**：

1. **质量体系缺失**
   - 无标准化测试流程，测试活动依赖个人经验
   - 缺乏测试用例库，每次版本测试从零开始
   - 无自动化测试，纯手工回归，效率低下

2. **业务复杂度高**
   - 多个应用模块并行开发，测试资源紧张
   - 新高考政策频繁调整，需求变更频繁
   - 数据准确性要求极高（涉及学生成绩、选科等关键信息）

3. **团队规模扩张**
   - 从3人测试团队扩展到8人
   - 新人缺乏培训体系，上手慢
   - 测试标准不统一，质量参差不齐

### 2.2 任务（Task）

**核心目标**：
1. 建立标准化的测试流程体系
2. 沉淀可复用的测试资产
3. 引入自动化测试提升效率
4. 保障业务高质量交付

**量化指标**：
- 测试用例覆盖率 ≥ 90%
- 核心功能自动化覆盖率 ≥ 80%
- 版本上线缺陷逃逸率 < 5%
- 测试周期缩短 30%

### 2.3 行动（Action）

建立缺陷生命周期管理：
```
新建 → 确认 → 分配 → 修复 → 验证 → 关闭
  ↓                        ↓
拒绝 ←────────────────── 重新打开
```

- P0：系统崩溃、数据丢失、核心功能不可用
- P1：主要功能异常、性能严重下降
- P2：次要功能异常、UI问题
- P3：优化建议、体验问题

**具体案例**：

```

- 多科目成绩组合

场景2：边界数据
- 满分、零分
- 小数点精度（保留1位、2位）
- 并列成绩处理

场景3：异常数据
- 缺考、缓考标记
- 成绩为空
- 成绩格式异常（字母、特殊字符）
```

1. 小数成绩精度丢失问题

### 5.2 关键业务概念

**新高考改革**：
- "3+3"模式：语数外3门必考 + 3门选考
- "3+1+2"模式：语数外3门必考 + 1门首选 + 2门再选
- 等级赋分：将原始分按比例转换为等级分

**走班制**：
- 学生根据选科结果到不同教室上课
- 需要考勤系统支持跨班级考勤

**智能排课**：
- 基于AI算法自动生成课表
- 需要考虑教师、教室、时间等多维度约束
